## 1. Context

사용자는 라운드 시작 시 아침 뉴스가 나오고, Papers, Please처럼 플레이어가 개장을 승인할 수 있는 구조를 원했다.  
또한 개장 전에 사용할 수 있는 카드가 있고, 개장 후에는 플레이어 대상 종목뿐 아니라 다른 종목도 뉴스에 기반해 움직이기를 원했다.

기존 장 시작 이벤트만으로는 문서를 읽고 개장한다는 의식적인 단계가 약했다.  
따라서 P0 라운드를 개장 전 단계와 장중 단계로 분리할 필요가 생겼다.

---

## 2. Decision

### 2.1 라운드는 개장 전 단계와 장중 단계로 나눈다

P0 라운드 흐름은 다음과 같다.

```text
개장 전 뉴스 표시
→ 시장 브리핑 확인
→ 개장 전 카드 최대 1장 선택
→ 개장 승인
→ 장중 진행
→ 마감 정산
```

### 2.2 개장 전 뉴스는 문서로 표시한다

개장 전 화면은 Papers, Please식 문서 감성을 반영해 다음을 표시한다.

1. 오늘의 주요 뉴스
2. 영향 범위: 시장, 섹터, 종목
3. 예상 효과
4. 대상 종목 프로필 요약
5. Today Condition
6. 목표 가격 밴드
7. 예산 조건

원작의 입국 심사, 여권, 국경, 캐릭터, 문구, 레이아웃은 사용하지 않는다.

### 2.3 개장 전 카드는 최대 1장 사용한다

P0에서는 개장 전 카드를 라운드당 최대 1장만 사용할 수 있다.

추천 P0 카드는 다음 4개다.

| 카드 | 효과 방향 |
| --- | --- |
| 사전 포지션 구축 | 예산 소모, 개장 직후 가격 압력 증가 |
| 관심 신호 배치 | 개인 참여도 증가, 감시도 소폭 증가 |
| 방어 자금 배정 | 급락 방어, 사용 가능 예산 감소 |
| 관망 | 효과 없음, 예산 보전 |

여러 장 조합은 P1 이후로 미룬다.

### 2.4 플레이어는 개장 승인으로 장중에 진입한다

개장 전 단계의 마지막 액션은 개장 승인이다.

UI 표현은 다음을 사용할 수 있다.

1. 개장 승인 버튼
2. 장 개시 도장
3. 프린터 출력
4. 경고음 또는 승인음

### 2.5 장중에는 대상 종목과 비플레이어 종목이 함께 움직인다

P0에서는 플레이어 대상 종목 1개를 상세 계산하고, 비플레이어 종목은 간략 등락률만 계산한다.

시장 보드는 다음을 표시한다.

1. 대상 종목
2. 같은 섹터 종목 2개
3. 다른 섹터 대표 종목 4~6개

### 2.6 뉴스 영향은 비플레이어 종목에도 반영한다

뉴스가 특정 섹터나 종목에 영향을 주면, 해당 대상은 시장 보드에서 실제로 더 크게 움직인다.

예를 들어 에너지·전력망 섹터 호재가 발생하면 에너지 섹터 종목의 등락률이 더 강하게 반응한다.

---

## 3. Consequences

### 3.1 Positive Consequences

1. 라운드 시작이 즉시 전투가 아니라 문서 판단으로 시작된다.
2. Papers, Please식 감성을 게임 구조와 직접 연결한다.
3. 개장 전 카드로 플레이어의 첫 선택이 생긴다.
4. 시장 보드 덕분에 내 종목만 움직이는 장난감 시장처럼 보이지 않는다.
5. 플레이어 종목과 무관한 뉴스도 시장 관심 분산으로 의미를 가질 수 있다.
6. 대상 종목만 상세 계산하므로 구현 복잡도를 제어할 수 있다.

### 3.2 Negative Consequences

1. 한 판 시작 전 UI 단계가 추가되어 진입 속도가 늦어질 수 있다.
2. 시장 보드 UI가 장중 화면을 복잡하게 만들 수 있다.
3. 비플레이어 종목 계산이라도 데이터와 튜닝 비용이 생긴다.
4. 개장 전 카드 밸런스가 약하면 무의미한 선택이 될 수 있다.

---

## 4. Alternatives Considered

### 4.1 즉시 장중 시작

게임 속도는 빠르지만 문서 감성이 약하다.  
사용자가 원하는 Papers, Please식 개장 경험과 맞지 않아 채택하지 않는다.

### 4.2 모든 24개 종목 상세 계산

시장감은 강하지만 P0 구현과 밸런싱 부담이 크다.  
대상 종목 상세 계산, 비플레이어 종목 간략 계산으로 제한한다.

### 4.3 개장 전 카드 여러 장 사용

전략성은 높지만 초반 선택 복잡도가 증가한다.  
P0에서는 최대 1장으로 제한한다.

---

## 5. Follow-ups

1. 개장 전 카드의 정확한 수치 효과는 SRS/SDD에서 확정한다.
2. 시장 보드에 표시할 비플레이어 종목 선정 규칙을 SRS/SDD에서 확정한다.
3. 뉴스 효과가 시장 관심 분산에 미치는 공식은 SRS/SDD에서 확정한다.
4. 개장 승인 연출은 P0 아트 방향과 함께 구체화한다.
