## 1. Context

PRD v0.1.2는 예산 시스템과 안전한 시장 압력 표현을 확정했지만, 매 라운드의 시작 조건이 반복적으로 느껴질 수 있었다.

사용자는 매일 장 시작 시 특정 시장 섹터나 종목에 대한 호재·악재 이벤트가 있기를 원했다.  
또한 농업, 에너지, 바이오처럼 현실감 있는 섹터가 있되, 실제 종목이나 실제 시장 데이터는 사용하지 않아야 했다.

P0의 목적은 복잡한 경제 시뮬레이션이 아니라, 매 판의 첫 판단을 다르게 만드는 작은 랜덤성과 문서형 브리핑 감성을 만드는 것이다.

---

## 2. Decision

### 2.1 P0 섹터는 8개로 고정한다

P0에서는 다음 8개 허구 섹터를 사용한다.

| 섹터 ID | 표시명 | 성격 |
| --- | --- | --- |
| food_agri | 식량·농업 | 안정적이지만 날씨·수출 이벤트 영향이 큼 |
| energy_grid | 에너지·전력망 | 호재/악재가 강하고 변동성이 큼 |
| bio_trial | 바이오·임상 | 고위험 고수익, 뉴스 이벤트 의존도가 큼 |
| automation_ai | 자동화·AI | 관심도 상승이 쉽고 과열·감시 위험이 큼 |
| chip_equipment | 칩·장비 | 추진력과 예산 효율이 좋지만 공급망 악재에 민감 |
| pay_fintech | 결제·핀테크 | 예산 효율은 좋지만 규제 이벤트에 취약 |
| media_games | 미디어·게임 | 관심도와 뉴스 티커 효과가 강함 |
| community_token | 커뮤니티 토큰 | 폭발력은 크지만 변동성·감시도·붕괴 위험이 큼 |

### 2.2 P0 종목은 섹터당 3개, 총 24개로 둔다

P0에서는 섹터별 허구 종목 3개를 제공한다.

예시는 다음과 같다.

| 섹터 | 허구 종목 예시 |
| --- | --- |
| 식량·농업 | 푸른창고, 작물망, 풍년그리드 |
| 에너지·전력망 | 볼트강, 전력나무, 태양제련 |
| 바이오·임상 | 바이타루프, 유전자둥지, 트라이얼폭스 |
| 자동화·AI | 오토마인드, 논리공장, 뉴럴항만 |
| 칩·장비 | 칩마당, 웨이퍼둥지, 신호제련 |
| 결제·핀테크 | 페이브릿지, 클리어민트, 장부철도 |
| 미디어·게임 | 하이프스테이지, 픽셀조합, 스트림둥지 |
| 커뮤니티 토큰 | 밈항구, 달연못, 군중발바닥 |

### 2.3 장 시작 이벤트는 시장, 섹터, 종목 범위를 가진다

P0에서는 매 라운드 시작 시 주요 이벤트 1개를 적용한다.

이벤트 범위는 다음 중 하나다.

1. 시장 전체
2. 특정 섹터
3. 특정 종목

이벤트 타입은 다음을 사용한다.

| 타입 | 설명 |
| --- | --- |
| 호재 | 관심도와 초기 가격 압력 증가 |
| 악재 | 하락 압력, 방어 비용, 변동성 증가 |
| 과열 | 상승 가능성과 감시도 증가를 동시에 제공 |
| 침체 | 관심도 또는 예산 효율 감소 |
| 규제 경고 | 감시도 시작값 또는 감시 증가량 증가 |
| 경쟁 출현 | 경쟁 압박 또는 저항 증가 |

### 2.4 실제 시장 데이터는 사용하지 않는다

P0에서는 다음을 사용하지 않는다.

1. 실제 종목명
2. 실제 거래소명
3. 실제 가격 데이터
4. 실제 뉴스 제목
5. 실제 사건 기반 이벤트
6. 실제 캘린더형 경제 일정

모든 섹터, 종목, 뉴스, 이벤트는 허구로 작성한다.

---

## 3. Consequences

### 3.1 Positive Consequences

1. 매 라운드의 시작 조건이 달라져 반복성이 줄어든다.
2. 뉴스 문서를 읽는 이유가 생긴다.
3. Papers, Please식 문서 브리핑과 게임 규칙이 연결된다.
4. 섹터별 성격이 있어 종목 선택이 의미를 갖는다.
5. 허구 섹터와 종목을 사용해 실제 금융 행위 재현 위험을 낮춘다.
6. 8개 섹터는 다양성과 구현 범위 사이에서 균형이 좋다.

### 3.2 Negative Consequences

1. 종목 데이터가 24개로 늘어 데이터 작성량이 증가한다.
2. 섹터별 밸런싱이 필요하다.
3. 뉴스가 반복되면 패턴이 빨리 보일 수 있다.
4. 실제 시장과 거리가 있어 현실 시뮬레이션으로 느껴지지 않을 수 있다.

---

## 4. Alternatives Considered

### 4.1 실제 산업 섹터 그대로 사용

현실감은 높지만 실제 시장과 연결되는 느낌이 강해진다.  
안전성과 풍자성을 위해 채택하지 않는다.

### 4.2 섹터 없이 종목만 랜덤 생성

구현은 단순하지만 뉴스와 시장 보드가 의미를 잃는다.  
섹터 기반 이벤트가 핵심 재미이므로 채택하지 않는다.

### 4.3 P0에서 12개 이상 섹터 제공

다양성은 높지만 이벤트와 밸런싱 범위가 과도하게 커진다.  
P0에서는 8개로 제한한다.

---

## 5. Follow-ups

1. 각 섹터의 기본 계수는 SRS/SDD에서 확정한다.
2. P0 아침 뉴스는 최소 10개 이상 작성한다.
3. 각 종목의 Asset Profile은 ADR-0004에 따른다.
4. 뉴스 적용 방식과 시장 보드 연동은 ADR-0005에 따른다.
