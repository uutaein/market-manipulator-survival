이 문서는 호가벽이 세워지고 녹는 과정을 더 부드럽고 읽기 쉬운 시각 피드백으로 보여주는 제품 요구를 정의한다.

---

## 0. 변경 이력

| 버전 | 변경 내용 |
| --- | --- |
| v0.2.4 | 호가창 depth 막대와 SIZE 숫자를 부드럽게 보간하고, 벽 생성/증가/소진 순간에만 짧은 펄스를 발생시키는 UI 품질 슬라이스를 추가한다. |

---

## 1. 제품 의도

v0.2.3에서 호가벽은 남은 잔량을 가지고 압력에 따라 줄어든다. 하지만 표시가 한 번에 바뀌거나 매 초 애니메이션이 재시작되면, 플레이어는 벽이 “버티고 녹는” 흐름보다 화면이 끊기는 느낌을 받는다.

v0.2.4의 목적은 다음이다.

1. 벽을 세우는 순간 depth 막대가 크게 차오르는 느낌을 준다.
2. 벽이 녹을 때 막대와 SIZE 숫자가 같은 방향으로 부드럽게 줄어든다.
3. 매 업데이트마다 애니메이션이 처음부터 반복되지 않게 한다.
4. 기존 hover-row 조작과 안전한 허구 depth 표현을 유지한다.

---

## 2. 범위

### 2.1 포함

| 항목 | 설명 |
| --- | --- |
| 막대 보간 | depth 막대는 layout width 변경이 아니라 transform 기반 스케일로 보간한다. |
| 숫자 보간 | SIZE 숫자는 기존 목표값으로 부드럽게 이동한다. |
| 전이 펄스 | 벽 활성화, depth 증가, depth 소진 때만 짧은 시각 펄스를 준다. |
| 안정적 행 클래스 | 매 tick마다 row class를 통째로 재설정해 애니메이션을 재시작하지 않는다. |
| 정리 로직 | overlay 제거 시 진행 중인 animation frame과 timeout을 정리한다. |

### 2.2 제외

| 항목 | 제외 이유 |
| --- | --- |
| 새 조작 버튼 | ADR-0036의 hover-row UX 유지 |
| 새 게임 규칙 | v0.2.3의 벽 소진 모델을 변경하지 않음 |
| 실제 시장 시각화 | 허구 게임 depth 표현만 개선 |

---

## 3. 플레이어 경험

1. 호가벽이 활성화되면 해당 행의 막대가 즉시 목표값을 향해 크게 차오른다.
2. 벽 잔량이 줄면 SIZE 숫자와 막대가 함께 줄어든다.
3. 활성 벽 행은 안정적인 강조 상태를 유지하되, 큰 변화 순간에만 짧게 빛난다.
4. hover 시 액션 affordance는 기존처럼 행 위에 표시된다.

---

## 4. SRS 위임 사항

SRS는 다음을 확정한다.

1. DOM row class 유지 규칙
2. depth 막대 보간 방식
3. SIZE 숫자 보간 방식
4. 전이 펄스 발생 조건
5. overlay destroy 정리 조건
