이 문서는 가상 호가벽이 고정된 가격 레벨에서 잔량을 잃고, 남은 예약금만 환급되는 플레이 경험을 정의한다.

---

## 0. 변경 이력

| 버전 | 변경 내용 |
| --- | --- |
| v0.2.3 | 활성 호가벽에 남은 depth와 남은 환급 예산을 추가한다. 반대 방향 압력과 벽 접촉은 벽 잔량을 줄이고, 가격 장벽은 벽이 녹기 전까지만 유지된다. 제거/만료 시에는 남은 예약금만 환급한다. |

---

## 1. 제품 의도

v0.2.1의 호가벽은 클릭한 줄의 SIZE를 즉시 키우고 가격 장벽을 만든다. v0.2.2는 이 표시 depth를 로컬 합성 체결 엔진으로 통과시키는 경계를 추가했다. v0.2.3에서는 벽이 압력을 받으면 잔량이 줄어드는 상태를 추가해, 벽이 버티는 동안과 녹아 사라지는 순간을 플레이어가 읽을 수 있게 한다.

이 기능의 목적은 다음이다.

1. 벽을 세운 가격 레벨의 대량 depth가 즉시 보이고, 이후 압력에 따라 점차 줄어들게 한다.
2. 매수벽은 남은 잔량이 있을 때만 해당 레벨 아래 하락을 막고, 매도벽은 남은 잔량이 있을 때만 해당 레벨 위 상승을 막는다.
3. 예약 예산은 벽 잔량이 녹은 만큼 소모되고, 제거 또는 만료 시 남은 예산만 환급된다.
4. 로컬 합성 체결 엔진은 유지하되, 이번 단계에서는 게임 상태 어댑터가 벽 잔량을 관리한다.

---

## 2. 범위

### 2.1 포함

| 항목 | 설명 |
| --- | --- |
| 벽 잔량 상태 | 활성 벽이 원래 depth와 남은 depth를 모두 가진다. |
| 환급 잔액 상태 | 예약 예산 중 아직 소모되지 않은 금액을 별도로 가진다. |
| 압력 기반 소진 | 벽 반대 방향의 가상 시장 압력과 가격 접촉이 벽 잔량을 줄인다. |
| 장벽 조건 | 벽 잔량이 남아 있는 동안에만 가격 장벽을 적용한다. |
| 표시 depth 연동 | 호가창 SIZE와 로컬 합성 depth snapshot은 남은 벽 잔량을 사용한다. |
| 환급 UX | 활성 벽을 뺄 때 현재 환급 가능한 금액을 보여준다. |
| 회귀 검증 | 부분 소진, 완전 소진, 환급, 장벽 해제, depth 반영을 검증한다. |

### 2.2 제외

| 항목 | 제외 이유 |
| --- | --- |
| 실제 거래소/실시장 체결 | 제품 안전 원칙상 제외 |
| 실제 주문 소진 로그 재현 | 이번 단계는 게임 상태 기반 소진 모델에 한정 |
| C++/WASM 교체 | ADR-0037의 후속 후보로 유지 |
| 같은 가격 레벨 중첩 | 기존처럼 가격 레벨당 하나의 활성 벽만 허용 |
| 별도 벽 관리 버튼 | 기존 호가 행 hover UX 유지 |

## 3. 플레이어 경험

1. 플레이어가 BID/ASK 행에서 벽을 세우면 해당 줄의 SIZE가 즉시 크게 증가한다.
2. 반대 방향 압력이 약하거나 가격이 벽에 닿지 않으면 벽은 대부분 유지된다.
3. 반대 방향 압력이 강하고 가격이 벽에 닿으면 SIZE가 눈에 띄게 줄어든다.
4. 벽 잔량이 남아 있으면 가격은 그 레벨을 넘지 못한다.
5. 벽 잔량이 모두 녹으면 가격 장벽이 사라지고, 같은 레벨은 cooldown만 남는다.
6. 활성 벽을 다시 클릭하면 남은 예약금만 환급된다.

---

## 4. 안전 표현 원칙

이 기능은 실제 거래 방법을 설명하거나 실제 시장 접근을 제공하지 않는다.

1. 모든 depth, 압력, 소진, 체결은 가상 게임 데이터다.
2. 실제 거래소명, 실제 티커, 실제 계좌, 실제 주문 API를 게임 엔진에 넣지 않는다.
3. 문서와 UI는 벽 소진을 “가상 depth 잔량”과 “게임 예산 예약/환급”으로만 표현한다.
4. 실제 시장에서의 주문 배치, 타이밍, 회피 절차를 다루지 않는다.

## 5. SRS 위임 사항

SRS는 다음을 확정한다.

1. 활성 벽 잔량 필드
2. 소진 입력과 산식
3. 남은 예약금 환급 규칙
4. 장벽 적용/해제 조건
5. 로컬 합성 depth snapshot 연동
6. UI 상태 문구
7. 회귀 검증 기준
