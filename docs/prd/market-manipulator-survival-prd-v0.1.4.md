### 0.1 현재 확정 기준

2026-06-13 기준 현재 제품 방향은 다음과 같다.

1. 게임은 TypeScript 기반의 경량 2D 브라우저 게임으로 만든다.
2. 1차 구현은 데스크톱 브라우저 실행을 우선한다.
3. 후속 버전에서 Electron 패키징을 검토할 수 있다.
4. 플레이어는 가상의 주식/코인 시장에서 활동하는 “운용 데스크” 역할을 맡는다.
5. 실제 종목, 실제 거래소, 실제 가격 데이터, 실제 투자 API는 사용하지 않는다.
6. 게임은 금융 범죄 방법을 설명하지 않고, 시장 압박을 추상화된 풍자 시스템으로 표현한다.
7. 핵심 플레이는 Vampire Survivors처럼 자동 발동되는 능력 조합, 웨이브 압박, 제한된 수동 대응을 따른다.
8. 화면과 UX는 Papers, Please처럼 문서, 도장, 감시 보고서, 제한 시간 선택, 관료제적 압박감을 중심으로 구성하되 원작 요소를 복제하지 않는다.
9. 일반적인 필드 맵 대신 차트, 뉴스 티커, 감시국 문서, 시장 보드, 작전 카드 패널이 전장 역할을 한다.
10. 한 판은 5일짜리 Run으로 만든다.
11. MVP는 싱글플레이어 오프라인 게임으로 만든다.
12. 결과 기록은 P0에서 localStorage에 저장한다.
13. P0 핵심 생존 자원은 **예산**이다.
14. 플레이어는 가격을 목표 밴드까지 끌어올리거나 유지하되, 예산을 일정 비율 이상 보전해야 완전 성공한다.
15. “자전거래”, “시세조종”, “허수 주문”처럼 실제 불법 행위를 직접 떠올리게 하는 표현은 UI와 사용자-facing 문구에서 사용하지 않는다.
16. 해당 개념은 “순환 유동성”, “시장 압력”, “관심 신호”, “가격 방어”, “과열 해소”, “포지션 정리” 같은 추상 게임 용어로 치환한다.
17. 각 Day는 항상 개장 전 뉴스 화면으로 시작한다.
18. 개장 전 단계에서 플레이어는 아침 뉴스와 시장 브리핑을 읽고, 개장 전 카드를 최대 1장 사용할 수 있다.
19. 플레이어가 개장 승인 버튼 또는 도장을 누르면 장중 단계로 전환된다.
20. P0에는 허구 섹터 8개와 섹터당 허구 종목 3개, 총 24개 종목을 둔다.
21. 모든 종목은 고정 Asset Profile을 가지며, Day마다 Today Condition만 ±10~15% 범위로 변동한다.
22. 장중에는 플레이어 대상 종목뿐 아니라 다른 종목의 등락률도 시장 보드에서 함께 움직인다.
23. 플레이어 대상 종목은 상세 계산하고, 비플레이어 종목은 뉴스·섹터·프로필·랜덤 변동성을 기반으로 간략 계산한다.
24. 장중 핵심 지표는 예산, 가격, 보유 비중, 개인 참여도, 시장 유동성, 감시도, 변동성이다.
25. 개인 참여도는 단순 수치가 아니라 차트 주변에 몰려오는 아이콘/토큰 기반 군집 웨이브로 시각화한다.
26. 각 Day의 장중 운용 시간은 MVP 기본값 6분으로 한다.
27. 개장 전 단계에는 시간 제한을 두지 않는다.
28. 즉시 실패 조건이 발생하면 해당 Day 정산을 기다리지 않고 Run 전체가 실패한다.
29. Day Settlement와 Final Settlement는 실제 수익과 감시등급을 함께 평가한다.
30. MVP 콘텐츠 수량은 PRD v0.1.4의 콘텐츠 동결 표를 기준으로 한다.
31. Morning News는 24개 이상의 수작업 뉴스가 아니라 5개 템플릿과 랜덤 허구 대상을 조합한다.
32. 장중 수동 액션은 `유동성 공급`, `가격 추진`, `과열 해소`, `포지션 정리` 4개로 고정한다.
33. MVP 개장 전 카드는 `시장 관찰`, `사전 포지션 구축`, `방어 자금 배정`, `관망` 4개로 고정한다.
34. MVP 장중 자동 카드는 `관심 신호`, `유동성 순환`, `가격 지지`, `변동성 흡수`, `뉴스 증폭`, `감시 완충`, `경쟁 견제`, `정리 루틴` 8개로 고정한다.
35. MVP 문서 이벤트는 `이상 흐름 질의서`, `시장 과열 경보`, `유동성 경색 보고`, `커뮤니티 폭주 알림`, `경쟁 데스크 개입 보고`, `급락 위험 통지`, `내부 리스크 메모`, `마감 전 정리 요청` 8개로 고정한다.
36. MVP Day 결과는 `완전 성공`, `위험 성공`, `고위험 성공`, `안정 운용`, `위험 운용`, `조용한 실패`, `손실 마감`, `강제 실패` 8개로 고정한다.
37. MVP Final 등급은 `S — 조용한 대성공`, `A — 성공적 운용`, `B — 위험한 성공`, `C — 간신히 생존`, `D — 실패한 운용`, `F — 강제 종료` 6개로 고정한다.
38. MVP Market Board는 총 8개 종목을 표시한다. 내 종목 1개는 상세 계산하고, 비플레이어 종목 7개는 간략 계산한다.
39. 보유 비중은 장중에는 영향력이지만 마감 정산에서는 리스크다. MVP에서는 마감 보유 비중을 4구간으로 분류한다.
40. MVP 화면은 `메인 메뉴`, `Run 시작 / 종목 선택`, `아침 뉴스 / 시장 브리핑`, `개장 전 카드 선택`, `장중 운용 화면`, `문서 이벤트 팝업`, `Day 정산 화면`, `Final 정산 화면` 8개로 제한한다.
41. MVP에서는 별도 튜토리얼 모드를 제공하지 않고, Day 1을 인게임 온보딩으로 설계한다.
42. 카드 시너지, 전체 시장 상세 시뮬레이션, 많은 뉴스, 복잡한 회계, 도감/통계, 플랫폼 확장은 MVP에서 제외한다.

## 1. 배경

주식과 코인 시장은 실제로 많은 사람에게 숫자, 차트, 뉴스, 소문, 군중 심리, 탐욕, 공포가 섞인 복잡한 공간으로 인식된다.

게임 관점에서 이 소재는 다음 이유로 흥미롭다.

1. 차트는 시간에 따라 계속 움직이는 자연스러운 전장이다.
2. 가격, 예산, 관심도, 감시도, 변동성은 서로 충돌하는 자원으로 만들기 쉽다.
3. 뉴스와 섹터 이벤트는 매 판 다른 시작 조건을 만든다.
4. 개인 참여 군집은 뱀서식 웨이브 압박을 시장 테마에 맞게 변환할 수 있다.
5. 문서, 도장, 브리핑, 감시 질의서는 Papers, Please식 압박감을 자연스럽게 만든다.

다만 이 소재에는 실제 금융 범죄를 설명하는 것처럼 보일 위험이 있다.  
따라서 본 제품은 실제 절차나 재현 가능한 방법을 다루지 않고, 모든 시장·종목·이벤트·카드 효과를 허구와 추상 수치로 표현한다.

## 2. 목표

이 제품의 목표는 **5일 Run 동안 매일 개장 전 뉴스 문서를 읽고, 개장 전 선택으로 그날의 장을 세팅한 뒤, 장중에는 시장 압력과 군중 심리를 실시간으로 조절해 실제 수익과 감시 리스크의 균형을 맞추는 경량 2D 풍자 게임**을 만드는 것이다.

### 2.1 핵심 목표

1. JS/TS 기반으로 가볍게 실행되는 2D 브라우저 게임을 만든다.
2. Vampire Survivors식 자동 카드 성장, 웨이브 압박, 제한된 수동 대응을 제공한다.
3. Papers, Please식 개장 전 문서 브리핑, 승인 도장, 감시 질의서, 제한 시간 선택 감성을 제공한다.
4. 실제 금융 행위가 아니라 허구의 시장 압력 조절 게임으로 표현한다.
5. 플레이어가 예산, 가격, 보유 비중, 개인 참여도, 시장 유동성, 감시도, 변동성을 동시에 관리하게 한다.
6. 매 Day 시작 시 아침 뉴스와 시장 브리핑으로 오늘의 섹터/종목 조건을 제시한다.
7. 개장 전 카드를 통해 장 시작 조건을 조정할 수 있게 한다.
8. 장중에는 대상 종목과 비대상 종목이 함께 움직이는 시장 보드를 제공한다.
9. 개인 참여도를 군집 웨이브로 시각화해 숫자가 아닌 화면 압박으로 체감하게 한다.
10. Day당 6분 장중 운용을 기본으로 하는 5일 Run MVP를 목표로 한다.

### 2.2 제품이 해결하려는 핵심 재미 문제

> 현재 게임의 핵심은 “가격을 올리는 것”이 아니라, **뉴스를 읽고 판을 세팅한 뒤, 과열과 감시, 군중 심리, 예산 고갈 사이에서 목표 밴드에 안착시키는 것**이다.

따라서 1차 제품은 다음 질문에 집중한다.

1. 오늘 아침 뉴스는 어느 섹터나 종목에 영향을 주는가?
2. 개장 전 어떤 카드를 사용할 것인가?
3. 내 종목은 원래 어떤 규모와 성향을 가진 종목인가?
4. 장중 군중이 몰려오는 흐름을 밀어야 하는가, 식혀야 하는가?
5. 예산을 얼마나 태우고 얼마나 보전할 것인가?
6. 목표 가격 밴드 안에서 마감할 수 있는가?
7. 감시도와 사회적 비용을 감당할 수 있는가?

## 3. Non-Goals

이번 1차 범위에서는 아래 기능을 포함하지 않는다.

1. 실제 주식/코인 API 연동
2. 실제 종목명, 실제 거래소명, 실제 가격 데이터 사용
3. 실제 시세조종 절차나 불법 금융 기법의 구체적 재현
4. 실제 주문 방식, 회피 방식, 거래 패턴 설명
5. 멀티플레이
6. 온라인 랭킹 서버
7. 현금 결제, 토큰, NFT, 실제 투자 기능
8. 모바일 최적화
9. 복잡한 경제 시뮬레이션 엔진
10. 모든 24개 종목의 상세 실시간 계산
11. 고해상도 실사 군중 애니메이션
12. 복잡한 유닛 AI
13. 실제 뉴스 크롤링 또는 캘린더형 경제 일정
14. 장중 복수 뉴스 체인
15. 클라우드 세이브
16. 리플레이 저장
17. 튜토리얼 외부 영상 제작

## 4. 핵심 게임 루프

MVP의 기본 루프는 다음과 같다.

```text
Run 시작
→ Day 1~5 반복
  → 대상 종목과 고정 Asset Profile 확인
  → Today Condition 적용
  → Morning News와 Market Briefing 확인
  → Pre-Open Card 최대 1장 선택 또는 관망
  → Opening Approval / 도장 액션
  → 6분 장중 운용
  → Day Settlement
→ Day 5 이후 Final Settlement
```

플레이어는 매 순간 다음 일곱 가지를 관리한다.

| 요소 | 의미 |
| --- | --- |
| 예산 | 카드 사용, 방어, 추진, 정산에 필요한 핵심 생존 자원 |
| 가격 | 목표 밴드에 안착시켜야 하는 대상 종목 지표 |
| 보유 비중 | 플레이어 데스크의 대상 종목 영향력 |
| 개인 참여도 | 군중 심리와 개인 참여 군집의 강도 |
| 시장 유동성 | 가격 반응성과 거래 흐름의 활발함 |
| 감시도 | 거래소/감시국 위험 |
| 변동성 | 가격이 통제 불가능하게 흔들리는 정도 |

### 4.1 v0.1.4 모호성 보고

v0.1.3과 ADR-0001~0008을 함께 검토한 결과, MVP 고정 전에 남아 있던 주요 모호성은 다음과 같다.

| 영역 | 기존 모호성 | v0.1.4 처리 |
| --- | --- | --- |
| 게임 단위 | 10분 단일 세션과 5일 Run 표현이 공존 | MVP는 5일 Run으로 고정. 각 Day의 장중 운용은 6분 |
| 콘텐츠 수량 | 뉴스, 카드, 이벤트, 결과 등급 수량이 일부 “추천” 또는 “후보”로 남음 | MVP 콘텐츠 동결 표로 기준 수량 지정. 이후 승인된 축소 결정으로 Morning News는 5개 템플릿, 수동 액션은 4개로 보정 |
| 정산 | 가격 목표, 실제 수익, 감시등급의 우선순위가 혼재 | 실제 수익과 감시등급을 이중 축으로 사용 |
| 보유 비중 | 영향력 지표인지 정산 리스크인지 불명확 | 장중 효율과 정산 리스크를 모두 만드는 핵심 지표로 정의 |
| Retail Swarm | 순수 시각화인지 실제 상태 변화와 연결되는지 모호 | 개인 참여도의 시각화이면서 가격/변동성/감시도에 연결되는 상태 장치로 정의 |
| 시장 보드 | 표시 종목 수와 전체 24개 종목 처리 범위가 불명확 | 총 8개 종목만 표시. 내 종목 1개는 상세 계산, 비플레이어 7개는 간략 계산 |
| 개장 전 단계 | 뉴스와 브리핑의 역할, 카드 생략 가능 여부, 승인 액션 필수 여부가 분산 | 뉴스는 조건 변화, 브리핑은 요약/판단 보조, 카드는 생략 가능, 승인은 필수 진입 액션 |

### 4.2 MVP 콘텐츠 동결

Resolved ambiguity: v0.1.4 초안과 ADR-0009/ADR-0010은 Morning News 15개 템플릿과 수동 액션 5개를 기준으로 했으나, 이후 승인된 MVP 축소 결정에 따라 Morning News는 5개 템플릿, 수동 액션은 4개로 변경한다. 24개 이상의 수작업 뉴스 콘텐츠와 `방어 자금 투입` 수동 버튼은 MVP에서 제외한다.

### 4.3 남은 PRD 결정 우선순위

SRS로 넘어가기 전에 우선적으로 닫아야 할 PRD 결정은 다음 순서다.

1. Day 간 이월 규칙의 PRD 수준 범위: 예산, 감시도, 보유 비중, 개인 참여도, 사회적 비용 중 무엇이 얼마나 추상적으로 이월되는지.
2. 24개 종목의 Asset Profile 배치 원칙: 모든 개별 수치가 아니라, 소형/중형/대형과 얇음/보통/두꺼움의 분포 기준.
3. Day 간 이월 규칙의 구체적 범위: 정산 결과가 다음 Day의 시작 예산, 감시도, 보유 비중, 뉴스 위험에 어떤 수준으로 반영되는지.
4. P1로 미룰 기능 목록 최종 정리: MVP에 절대 넣지 않을 확장 화면/시스템 확정.
5. Day 간 이월 규칙의 SRS 준비 수준: PRD에 남길 방향과 SRS로 넘길 수치의 경계.

### 5.1 Sector

Sector는 허구 시장의 산업·테마 분류다.

P0 섹터는 8개로 고정한다.

| 섹터 ID | 표시명 | 성격 |
| --- | --- | --- |
| food_agri | 식량·농업 | 안정적이지만 날씨·수출 이벤트 영향이 큼 |
| energy_grid | 에너지·전력망 | 호재/악재가 강하고 변동성이 큼 |
| bio_trial | 바이오·임상 | 고위험 고수익, 뉴스 이벤트 의존도가 큼 |
| automation_ai | 자동화·AI | 관심도 상승이 쉽고 과열·감시 위험이 큼 |
| chip_equipment | 칩·장비 | 추진력과 예산 효율이 좋지만 공급망 악재에 민감 |
| pay_fintech | 결제·핀테크 | 예산 효율은 좋지만 규제 이벤트에 취약 |
| media_games | 미디어·게임 | 관심도와 뉴스 티커 효과가 강함 |
| community_token | 커뮤니티 토큰 | 폭발력은 크지만 변동성·감시도·붕괴 위험이 큼 |

### 5.2 Asset

Asset은 플레이어가 대상 종목으로 선택하거나 시장 보드에서 관찰하는 허구 종목이다.

P0에서는 섹터당 3개, 총 24개 종목을 둔다.

예시 종목은 다음과 같다.

| 섹터 | 허구 종목 예시 |
| --- | --- |
| 식량·농업 | 푸른창고, 작물망, 풍년그리드 |
| 에너지·전력망 | 볼트강, 전력나무, 태양제련 |
| 바이오·임상 | 바이타루프, 유전자둥지, 트라이얼폭스 |
| 자동화·AI | 오토마인드, 논리공장, 뉴럴항만 |
| 칩·장비 | 칩마당, 웨이퍼둥지, 신호제련 |
| 결제·핀테크 | 페이브릿지, 클리어민트, 장부철도 |
| 미디어·게임 | 하이프스테이지, 픽셀조합, 스트림둥지 |
| 커뮤니티 토큰 | 밈항구, 달연못, 군중발바닥 |

### 5.3 Asset Profile

Asset Profile은 종목의 고정 기본 성향이다.

각 종목은 다음 값을 가진다.

| 값 | 의미 |
| --- | --- |
| 규모 | 소형, 중형, 대형 |
| 유통 두께 | 얇음, 보통, 두꺼움 |
| 기본 관심도 | 시작 개인 참여도와 뉴스 반응성 |
| 기본 변동성 | 가격 흔들림 정도 |
| 감시 민감도 | 감시도 증가 보정 |
| 경쟁 강도 | 경쟁 압박 등장 가능성 |
| 예산 효율 | 예산 1당 시장 압력 효율 |
| 목표 난이도 | 목표 밴드 산정 난이도 |

### 5.4 Today Condition

Today Condition은 Day마다 Asset Profile에 적용되는 작은 변동이다.

P0에서는 일부 수치만 ±10~15% 범위에서 변동한다.  
완전 랜덤 프로필은 사용하지 않는다.

### 5.5 Morning News

Morning News는 개장 전 화면에 표시되는 오늘의 주요 뉴스다.

뉴스는 시장 전체, 특정 섹터, 특정 종목에 영향을 줄 수 있다.  
실제 뉴스나 실제 시장 데이터는 사용하지 않는다.
MVP에서는 5개 뉴스 템플릿과 랜덤 허구 대상을 조합해 매 Day 1개의 Morning News를 생성한다.

### 5.6 Pre-Open Card

Pre-Open Card는 개장 전 단계에서 최대 1장 사용할 수 있는 준비 카드다.
플레이어는 카드를 사용하지 않고 관망으로 개장할 수 있다.

예시는 다음과 같다.

| 카드 | 역할 |
| --- | --- |
| 시장 관찰 | 뉴스 효과와 목표 조건을 더 명확히 보여주는 정보 카드 |
| 사전 포지션 구축 | 시작 예산 일부 소모, 개장 직후 가격 압력 증가 |
| 방어 자금 배정 | 시작 예산 일부를 방어용으로 묶어 급락/패닉 리스크 완화 |
| 관망 | 카드 없이 개장, 예산 보전 |

### 5.7 Market Board

Market Board는 대상 종목 외 다른 종목의 등락률을 보여주는 보조 시장 화면이다.

P0에서는 다음을 표시한다.

1. 플레이어 대상 종목 1개
2. 같은 섹터 비플레이어 종목 2개
3. 다른 섹터 대표 종목 5개

화면에는 총 8개 종목을 표시한다.  
내 종목 1개는 상세 계산하고, 비플레이어 종목 7개는 간략 계산한다.

24개 종목 전체의 실시간 상세 시뮬레이션은 MVP에서 제외한다.

### 5.8 Retail Swarm

Retail Swarm은 개인 참여도를 시각화하는 아이콘/토큰 기반 군집 웨이브다.

개인 참여도가 높을수록 군집 수, 속도, 밀도가 증가한다.  
과열 상태에서는 군집이 밀집되고 감시도와 변동성이 함께 오른다.  
패닉 상태에서는 군집이 반대 방향으로 빠르게 이탈하며 가격 하락 압력을 만든다.
Retail Swarm은 순수 배경 연출이 아니라 개인 참여도의 상태 표현이며, 가격 압력, 변동성, 감시도 변화와 연결된다.

## 6. 핵심 사용자 시나리오

### 6.1 아침 뉴스 확인

사용자는 Day 시작 시 개장 전 시장 브리핑 문서를 확인한다.  
브리핑에는 오늘의 뉴스, 영향 범위, 적용 효과, 주의사항이 표시된다.

### 6.2 개장 전 카드 선택

사용자는 개장 전 카드 중 최대 1장을 선택한다.  
강한 준비 카드는 시작 예산을 소모하거나 감시도 위험을 높일 수 있다.

### 6.3 개장 승인

사용자는 도장 또는 개장 승인 버튼을 눌러 장중 단계로 진입한다.

### 6.4 장중 시장 관찰

사용자는 대상 종목 차트와 시장 보드를 함께 본다.  
대상 종목은 상세 계산되고, 다른 종목은 간략 등락률로 움직인다.

### 6.5 개인 참여 군집 대응

사용자는 차트 주변에 몰려오는 개인 참여 군집을 보고 현재 시장 과열도를 판단한다.  
군집이 과도하게 몰리면 과열 해소나 포지션 정리를 고려한다.

### 6.6 장중 액션 사용

사용자는 자동 카드와 제한된 수동 액션으로 유동성 공급, 가격 추진, 과열 해소, 포지션 정리를 수행한다.  
액션명은 실제 금융 행위가 아니라 추상화된 게임 용어로 표현한다.

### 6.7 Day Settlement와 Final Settlement

각 Day의 6분 장중 운용이 끝나면 목표 밴드, 실제 수익, 예산 보전율, 감시등급, 보유 비중, 변동성, 사회적 비용을 기준으로 Day Settlement가 표시된다.

Day 5 정산 후에는 누적 실제 수익, 최종 감시등급, 평균 감시등급, 목표 달성 Day 수, 최종 예산, 최종 보유 비중, 사회적 비용을 기준으로 Final Settlement가 표시된다.

## 7. 핵심 기능

### 7.1 개장 전 뉴스 화면

각 Day는 개장 전 뉴스 화면으로 시작한다.

화면 구성은 다음과 같다.

| 영역 | 설명 |
| --- | --- |
| 상단 | Run Day/날짜/대상 종목/섹터 |
| 중앙 | 아침 뉴스 문서, 시장 브리핑 |
| 우측 | 대상 종목 Asset Profile 요약 |
| 하단 | 개장 전 카드 선택, 개장 승인 버튼 |
| 보조 | Today Condition, 목표 밴드, 예산 조건 |

MVP에는 Morning News 템플릿 5개를 둔다.  
각 Day에는 활성 Morning News 1개를 적용한다.

Morning News의 역할은 해당 Day의 조건 변화를 만드는 것이다.  
Market Briefing의 역할은 플레이어가 읽고 판단할 수 있도록 뉴스 영향, 대상 종목 상태, 목표 밴드, 예산 조건, 주요 리스크를 요약하는 것이다.

### 7.1.1 MVP 아침 뉴스 템플릿

MVP에서는 아침 뉴스 콘텐츠를 5개 템플릿으로 제한한다.

각 Day에는 1개의 아침 뉴스가 표시된다.  
뉴스는 고정 콘텐츠 24개 이상을 제작하는 대신, 5개의 뉴스 템플릿과 랜덤 적용 대상을 조합해 반복감을 줄인다.

MVP 뉴스 템플릿은 다음 5종이다.

| 템플릿 | English design label | PRD 역할 |
| --- | --- | --- |
| 섹터 호재 | Sector Positive Catalyst | 한 섹터의 관심도와 상승 시장 압력을 높임 |
| 섹터 악재 | Sector Negative Catalyst | 한 섹터의 하방 압력과 변동성을 높임 |
| 시장 침체 | Market Slump | 전체 관심도와 예산 효율을 낮춤 |
| 규제 경고 | Regulatory Warning | 시작 감시도 또는 감시 민감도를 높임 |
| 과열 확산 | Overheat Spread | 개인 참여도, 변동성, 감시 리스크를 높임 |

뉴스는 시장 전체, 특정 섹터, 특정 종목 중 하나를 대상으로 적용될 수 있다.  
단, MVP에서는 섹터 단위 영향을 우선한다.

예시는 다음과 같다.

```text
Template: 섹터 호재
Target: 식량·농업
Result: 식량·농업 섹터에 영향을 주는 허구의 긍정 이벤트
```

24개 이상의 수작업 뉴스 콘텐츠는 MVP 범위에서 제외하고, P1 이후 확장 후보로 둔다.

### 7.2 개장 전 카드

MVP의 개장 전 카드는 4개로 제한한다.

각 Day에서 플레이어는 아침 뉴스와 시장 브리핑을 확인한 뒤, 개장 전 카드 중 최대 1장을 선택할 수 있다.  
카드를 사용하지 않는 선택은 `관망`으로 표현한다.

MVP 개장 전 카드는 다음 4개다.

| 카드 | 효과 방향 |
| --- | --- |
| 시장 관찰 | 뉴스 효과와 목표 조건을 더 명확히 보여주는 정보 카드 |
| 사전 포지션 구축 | 시작 예산을 소모해 개장 직후 가격 추진력을 얻는 공격적 카드 |
| 방어 자금 배정 | 시작 예산 일부를 방어용으로 묶어 급락/패닉 리스크를 낮추는 방어 카드 |
| 관망 | 아무 카드도 쓰지 않고 예산을 보전한 채 개장하는 선택 |

개장 전 카드는 아침 뉴스에 대한 첫 대응 선택지 역할을 한다.  
플레이어는 정보를 더 확인할지, 개장 직후 추진력을 얻을지, 리스크를 줄일지, 아무것도 하지 않고 예산을 보전할지 선택한다.

개장 전 카드의 효과 방향은 개장 승인 전에 표시한다.  
정확한 수치와 계수는 SRS/SDD에서 정의한다.

카드 선택 후 플레이어는 `개장 승인` 버튼 또는 도장 액션을 통해 장중 단계로 진입한다.

### 7.3 개장 승인

개장 전 단계의 마지막 필수 액션은 개장 승인이다.

UI는 도장, 승인 버튼, 프린터 출력, 경고음 등으로 연출한다.  
개장 승인 후에는 장중 화면으로 전환되고 가격 계산이 시작된다.

### 7.4 종목 프로필

각 종목은 고정 Asset Profile을 가진다.

규모는 다음 3단계다.

| 규모 | 특징 |
| --- | --- |
| 소형 | 적은 예산으로 잘 움직이지만 변동성·감시 민감도가 높음 |
| 중형 | 표준형 |
| 대형 | 움직이기 어렵지만 안정적이고 감시 민감도가 낮음 |

유통 두께는 다음 3단계다.

| 유통 두께 | 특징 |
| --- | --- |
| 얇음 | 가격 반응이 크지만 급락 위험이 높음 |
| 보통 | 표준형 |
| 두꺼움 | 많은 예산이 필요하지만 안정적 |

### 7.5 Today Condition

Day 시작 시 Today Condition을 적용한다.

Today Condition은 종목의 고정 프로필 중 일부 수치를 ±10~15% 범위에서 변동한다.

예시는 다음과 같다.

```text
오늘의 컨디션:
관심도 +6
변동성 -3
예산 효율 -4%
```

### 7.6 시장 보드

장중에는 대상 종목 외 다른 종목의 등락률도 함께 표시한다.

MVP 시장 보드는 총 8개 종목을 표시한다.  
내 종목 1개는 상세 계산하고, 비플레이어 종목 7개는 간략 계산한다.

| 표시 대상 | 수량 |
| --- | ---: |
| 플레이어 대상 종목 | 1 |
| 같은 섹터 비플레이어 종목 | 2 |
| 다른 섹터 대표 종목 | 5 |

P0 계산 방식은 다음과 같다.

| 종목 유형 | 계산 방식 |
| --- | --- |
| 대상 종목 | 상세 계산. 예산, 카드, 보유 비중, 개인 참여도, 시장 유동성, 감시도, 변동성 모두 반영 |
| 비플레이어 종목 | 간략 계산. 뉴스, 섹터, Asset Profile, 랜덤 변동성만 반영 |

24개 종목 전체의 실시간 상세 시뮬레이션은 MVP에서 제외한다.

비플레이어 종목은 플레이어가 직접 조작하지 않는다.

비플레이어 종목의 표시 정보는 종목명, 섹터, 현재 등락률, 뉴스 영향 배지 또는 간단한 상태 표시, 과열/패닉 상태, 간략 추세 아이콘으로 제한한다.

### 7.7 장중 인터랙션

장중 단계에서 플레이어는 다음 지표를 실시간으로 관리한다.

| 지표 | 역할 |
| --- | --- |
| 예산 | 카드 사용 비용, 실패 조건, 정산 기준 |
| 가격 | 목표 밴드 판정 기준 |
| 보유 비중 | 대상 종목 영향력 |
| 개인 참여도 | Retail Swarm과 가격 추진 보정 |
| 시장 유동성 | 가격 반응성과 변동성 |
| 감시도 | 100 도달 시 실패 |
| 변동성 | 랜덤 흔들림과 급락 위험 |

P0 장중 액션은 다음 범주로 구성한다.

| 범주 | 추상 표현 | 역할 |
| --- | --- | --- |
| 유동성 | 유동성 공급 / 순환 유동성 | 거래 흐름과 가격 반응 증가 |
| 추진 | 가격 추진 | 상승 압력 생성 |
| 안정화 | 과열 해소 / 가격 안정화 | 감시도와 변동성 완화 |
| 정산 | 포지션 정리 | 예산 회복, 시장 압력 감소 |

### 7.8 자동 카드와 수동 액션

P0 장중 조작은 자동 카드와 제한된 수동 액션을 조합한다.

1. 자동 카드는 레벨업 또는 선택을 통해 획득하며 일정 주기로 발동한다.
2. 수동 액션은 쿨다운과 예산 비용을 가진다.
3. P0 자동 카드는 8개로 제한한다.
4. P0 수동 액션은 4개로 제한한다.
5. 모든 효과 설명은 수치 중심으로 표시한다.

### 7.8.1 MVP 장중 수동 액션

MVP의 장중 수동 액션은 4개로 제한한다.

| 액션 | 역할 |
| --- | --- |
| 유동성 공급 | 시장 유동성과 가격 반응성을 높인다. 예산을 소모하고 감시도 또는 변동성을 높일 수 있다. |
| 가격 추진 | 가격을 목표 밴드 방향으로 밀어준다. 강하지만 위험하다. |
| 과열 해소 | 가격, 개인 참여도, 변동성, 감시 리스크를 안정화한다. 상승 압력을 낮출 수 있다. |
| 포지션 정리 | 보유 비중을 낮추고 예산을 회복한다. 가격 지지력이 약해지거나 단기 불안정이 늘 수 있다. |

MVP에서 수동 버튼으로 제공하는 액션은 위 4개로 고정한다.

수동 액션은 플레이어가 장중 상황에 즉시 대응하기 위한 최소 조작 세트다.

`방어 자금 투입`, `군중 진정`, `관심 신호`는 MVP에서는 수동 버튼이 아니다.  
이들은 자동 카드, 문서 이벤트 효과, 또는 P1 후보로 둔다.

### 7.8.2 MVP 장중 자동 카드

MVP의 장중 자동 카드는 8개로 제한한다.

자동 카드는 플레이어가 직접 누르는 수동 액션과 달리, 장중에 주기적으로 발동되거나 지속 효과를 제공하는 빌드 요소다.  
플레이어는 장중 일정 조건마다 자동 카드 선택지를 받고, 새 카드를 획득하거나 기존 카드를 강화할 수 있다.

MVP 자동 카드는 다음 8개다.

| 자동 카드 | 담당 지표 | 효과 방향 | PRD 역할 |
| --- | --- | --- | --- |
| 관심 신호 | 개인 참여도 | 개인 참여도 증가, 가격 추진 효율 증가, 변동성/감시 리스크 증가 가능 | 시장 관심을 끌어 가격 반응성을 높이는 성장 카드 |
| 유동성 순환 | 시장 유동성 | 시장 유동성 증가, 가격 반응성 증가, 예산 소모 또는 감시 리스크 증가 가능 | 장중 시장 흐름을 활발하게 만들어 가격이 더 잘 움직이게 하는 카드 |
| 가격 지지 | 가격 방어 | 하락 압력 완화, 급락 위험 감소, 예산 소모 가능 | 악재나 패닉 상황에서 가격 붕괴를 막는 방어 카드 |
| 변동성 흡수 | 변동성 | 변동성 증가 억제, 패닉 위험 감소, 가격 추진력 일부 감소 가능 | 과도한 흔들림을 줄여 목표 밴드 안착을 돕는 안정화 카드 |
| 뉴스 증폭 | 뉴스 효과 | 현재 Day의 아침 뉴스 효과 일부 강화. 호재는 추진력, 악재는 악재 영향도 커질 수 있음 | 그날의 시장 분위기에 올라타는 고위험/고보상 카드 |
| 감시 완충 | 감시도 | 감시도 증가량 완화, 강한 액션의 리스크 감소, 가격 추진력 없음 | 고위험 운용을 조금 더 오래 버티게 해주는 방어적 카드 |
| 경쟁 견제 | 경쟁 압박 | 경쟁 압박 감소, 시장 저항 감소, 일부 감시 리스크 가능 | 다른 시장 압력이나 경쟁 데스크의 방해를 줄이는 카드 |
| 정리 루틴 | 보유 비중/예산 회수 | 보유 비중 과다 상태 완화, 예산 회수 보조, 가격 지지 약화 가능 | 마감 전에 과도한 보유 비중을 줄이고 정산 리스크를 낮추는 카드 |

자동 카드와 수동 액션의 차이는 다음과 같다.

| 구분 | 역할 |
| --- | --- |
| 수동 액션 | 플레이어가 즉시 대응하는 버튼 |
| 자동 카드 | 빌드 선택에 따라 주기적으로 발동되는 지속 효과 |

예시는 다음과 같다.

```text
수동 액션: 지금 가격을 밀어야겠다 → 가격 추진 클릭
자동 카드: 일정 주기마다 관심 신호가 발동되어 개인 참여도가 조금씩 오른다
```

각 자동 카드는 Lv.1~Lv.3까지 성장할 수 있다.  
MVP에서는 레벨업 시 효과량 증가 또는 발동 주기 단축 중 하나만 적용한다.  
복잡한 분기 업그레이드, 카드 진화, 조합 시너지는 P1 이후로 둔다.

MVP의 자동 카드 획득 방식은 다음 방향으로 둔다.

1. Day 시작 시 기본 자동 카드 1개를 보유한다.
2. 장중 일정 조건마다 카드 선택 보상이 발생한다.
3. 선택지는 3개로 제한한다.
4. 선택한 카드는 새 카드 획득 또는 기존 카드 레벨업으로 처리한다.

정확한 보상 발생 조건, 발동 주기, 효과량, 레벨별 수치는 SRS/SDD에서 확정한다.

### 7.9 MVP 문서 이벤트

MVP의 문서 이벤트는 8개로 제한한다.

문서 이벤트는 장중에 조건 기반으로 발생하는 돌발 의사결정이다.  
이벤트는 거래소 질의서, 시장 과열 경보, 내부 리스크 메모, 커뮤니티 알림 같은 문서 UI로 표시된다.

MVP에서는 문서 이벤트 발생 시 장중 시간이 일시정지된다.  
플레이어가 선택지를 고르면 효과가 적용되고 장중 시간이 재개된다.

각 문서 이벤트는 3개의 선택지를 가진다.  
선택지는 대체로 안정 선택지, 공격 선택지, 회피/관망 선택지로 구성된다.

| 선택지 성격 | 방향 |
| --- | --- |
| 안정 선택지 | 감시도/변동성 감소, 예산 또는 가격 추진력 손실 |
| 공격 선택지 | 가격/수익 기회 증가, 감시도/변동성 증가 |
| 회피/관망 선택지 | 즉시 비용 없음, 후속 리스크 증가 |

MVP 문서 이벤트는 다음 8개다.

| 이벤트 | 등장 조건 방향 | 역할 |
| --- | --- | --- |
| 이상 흐름 질의서 | 감시도가 높을 때 | 감시 리스크를 직접 다루는 이벤트 |
| 시장 과열 경보 | 개인 참여도 또는 변동성이 높을 때 | 너무 뜨거운 시장을 식힐지, 계속 밀지 선택 |
| 유동성 경색 보고 | 예산 또는 시장 유동성이 낮을 때 | 자원이 부족할 때 발생하는 압박 이벤트 |
| 커뮤니티 폭주 알림 | 개인 참여도 과열 또는 Retail Swarm 밀집 시 | 군중 심리를 어떻게 다룰지 선택 |
| 경쟁 데스크 개입 보고 | 경쟁 압박이 높을 때 | 다른 시장 압력과 경쟁 구도를 표현 |
| 급락 위험 통지 | 가격이 붕괴선에 가까울 때 | 실패 직전의 긴급 선택 |
| 내부 리스크 메모 | 보유 비중이 너무 높을 때 | 보유 비중 리스크를 정산 전에 인지시키는 이벤트 |
| 마감 전 정리 요청 | Day 후반부 또는 마감 직전 | 하루 정산 전에 마지막 전략 선택 유도 |

이벤트별 선택지는 SRS에서 구체화하되, PRD 기준 역할은 다음 방향을 따른다.

| 이벤트 | 안정 선택지 방향 | 공격 선택지 방향 | 회피/관망 선택지 방향 |
| --- | --- | --- | --- |
| 이상 흐름 질의서 | 예산 소모, 감시도 감소 | 가격 추진력 감소, 감시도 감소 | 즉시 비용 없음, 감시도 증가 |
| 시장 과열 경보 | 가격 압력 감소, 감시도/변동성 감소 | 가격 압력 유지, 감시도 증가 | 단기 가격 보너스, 큰 리스크 |
| 유동성 경색 보고 | 일부 예산 회복, 가격 압력 감소 | 예산 소모, 유동성 증가 | 비용 없음, 변동성 증가 가능 |
| 커뮤니티 폭주 알림 | 개인 참여도 감소, 변동성 감소 | 가격 추진력 증가, 감시도 증가 | 랜덤 결과, 패닉 위험 증가 |
| 경쟁 데스크 개입 보고 | 예산 소모, 경쟁 압박 감소 | 예산 소모, 하락 압력 감소 | 비용 없음, 경쟁 압박 증가 |
| 급락 위험 통지 | 예산 소모, 급락 압력 감소 | 예산 회복, 가격 지지 약화 | 비용 없음, 붕괴 위험 증가 |
| 내부 리스크 메모 | 보유 비중 감소, 예산 회복 | 가격 추진 효율 유지, 감시도 증가 | 즉시 변화 적음, 정산 패널티 가능 |
| 마감 전 정리 요청 | 보유 비중 감소, 감시도 감소, 수익 일부 제한 | 가격 압력 증가, 감시도/변동성 증가 | 비용 없음, 현재 상태로 마감 |

문서 이벤트는 Papers, Please식 문서 판단과 장중 시장 압박을 연결하는 장치다.  
정확한 등장 조건, 선택지 수치, 발생 쿨다운, 중복 방지 규칙은 SRS/SDD에서 확정한다.

### 7.10 보유 비중

보유 비중은 플레이어 데스크가 대상 종목에 갖는 영향력 지표다.

핵심 원칙은 다음과 같다.

> 보유 비중은 장중에는 힘이지만, 마감 때는 리스크다.

장중에는 보유 비중이 높을수록 다음 효과가 발생한다.

1. 가격 추진 효율 증가
2. 가격 방어 효율 증가
3. 포지션 정리 시 예산 회복량 증가
4. 감시도 증가량 증가

MVP에서 보유 비중은 수동과 자동 양쪽으로 변할 수 있다.  
수동 액션 중에서는 `포지션 정리`가 보유 비중을 낮추는 명시적 조작이다.  
개장 전 카드와 자동 카드는 보유 비중 또는 시장 영향력을 간접적으로 올리거나 낮출 수 있으나, 실제 금융 절차처럼 표현하지 않는다.

MVP에서는 마감 보유 비중을 4구간으로 분류한다.

| 구간 | 보유 비중 | 정산 의미 |
| --- | ---: | --- |
| 영향력 부족 | 0~10% | 가격 영향력과 수익 기여가 낮음 |
| 안정 구간 | 10~35% | 정상 정산 구간 |
| 부담 구간 | 35~55% | 수익 잠재력은 높지만 감시/사회적 비용 증가 |
| 과점 위험 | 55% 이상 | 정산 패널티가 크고 감시등급 악화 가능 |

포지션 정리는 보유 비중을 낮추고 예산을 회수하는 핵심 수동 액션이다.  
플레이어는 장중에 가격을 목표 밴드로 이동시키는 것뿐 아니라, 마감 전에 과도한 보유 비중을 줄일지도 판단해야 한다.

### 7.11 개인 참여도

개인 참여도는 군중 심리와 Retail Swarm 밀도를 나타낸다.

| 개인 참여도 | 상태 | 효과 방향 |
| --- | --- | --- |
| 0~25 | 무관심 | 가격 추진 어려움 |
| 26~60 | 관심 형성 | 정상 |
| 61~85 | 과열 | 가격 추진 쉬움, 감시도 증가 |
| 86~100 | 광풍 | 급등락, 감시 이벤트, 붕괴 위험 |

### 7.12 Retail Swarm 시각화

장중 단계에서 개인 참여도는 단순 수치뿐 아니라 시각적 군집 웨이브로 표현한다.

P0 구현 기준은 다음과 같다.

1. 아이콘 또는 토큰 기반의 추상 군집을 사용한다.
2. 개인 참여도가 높을수록 스폰 수와 이동 속도가 증가한다.
3. 과열 상태에서는 군집 밀도가 높아지고 경고 색/깜빡임을 사용한다.
4. 패닉 상태에서는 군집이 반대 방향으로 빠르게 이탈한다.
5. Retail Swarm은 가격, 변동성, 감시도 변화와 연결된다.
6. 실사형 군중 묘사와 복잡한 유닛 AI는 P0에 포함하지 않는다.

Retail Swarm 상태는 MVP에서 3종으로 시작한다.

| 상태 | 효과 방향 |
| --- | --- |
| 관심 | 개인 참여도와 가격 추진 보너스 증가 |
| 과열 | 가격 급등 가능, 감시도와 변동성 증가 |
| 패닉 | 가격 하락 압력과 변동성 증가 |

### 7.13 가격 계산 방향

P0 가격 계산은 실제 금융 모델이 아니라 게임용 Tick Formula로 구현한다.

대상 종목 가격 변화는 다음 요소를 반영한다.

```text
가격 변화 =
기본 랜덤 변동
+ 시장 압력
+ 뉴스 보정
+ 개인 참여도 보정
+ 보유 비중 보정
+ 시장 유동성 보정
- 시장 저항
± 변동성 흔들림
```

비플레이어 종목 가격 변화는 더 단순하게 계산한다.

```text
비플레이어 종목 가격 변화 =
기본 랜덤 변동
+ 뉴스 보정
+ 섹터 보정
± 변동성 흔들림
```

정확한 수식과 계수는 SRS/SDD 단계에서 확정한다.

## 9. 비즈니스 규칙

### 9.1 안전 표현 규칙

1. 실제 종목, 실제 거래소, 실제 가격 데이터, 실제 사건은 사용하지 않는다.
2. 실제 금융 범죄 절차를 단계적으로 설명하지 않는다.
3. UI와 튜토리얼에서는 직접적인 불법 행위 표현을 사용하지 않는다.
4. 카드 효과는 게임 수치 중심으로만 설명한다.
5. 결과 정산에는 사회적 비용과 감시 리스크를 함께 표시한다.

### 9.2 개장 전 규칙

1. 모든 Day는 개장 전 뉴스 화면에서 시작한다.
2. P0에서는 Day당 주요 뉴스 1개를 적용한다.
3. MVP Morning News는 5개 템플릿과 랜덤 허구 대상을 조합한다.
4. 뉴스는 시장, 섹터, 종목 중 하나의 범위를 가질 수 있다.
5. MVP에서는 섹터 단위 영향을 기본 우선순위로 둔다.
6. 플레이어는 개장 전 카드를 최대 1장 사용할 수 있다.
7. 카드 없이 관망하고 개장할 수 있다.
8. 개장 승인 후 장중 단계로 전환된다.

### 9.3 종목 프로필 규칙

1. 모든 종목은 고정 Asset Profile을 가진다.
2. P0에서는 완전 랜덤 종목 프로필을 사용하지 않는다.
3. Today Condition은 일부 수치만 ±10~15% 범위로 조정한다.
4. 목표 밴드와 시작 예산은 종목 규모와 목표 난이도를 기반으로 산정한다.

### 9.4 시장 보드 규칙

1. 장중에는 대상 종목 외 다른 종목도 등락률을 표시한다.
2. 대상 종목은 상세 계산한다.
3. MVP 시장 보드는 총 8개 종목을 표시한다.
4. 비플레이어 종목 7개는 간략 계산한다.
5. 24개 종목 전체의 실시간 상세 시뮬레이션은 MVP에서 제외한다.
6. 비플레이어 종목은 플레이어가 직접 조작할 수 없다.
7. 뉴스의 영향을 받는 섹터나 종목은 시장 보드에서 배지 또는 간단한 상태 표시로 드러나야 한다.

### 9.5 예산 규칙

1. 시작 예산은 P0 기본값 100을 기준으로 하되, 종목 규모에 따라 보정할 수 있다.
2. 최소 유지 예산은 P0 기본값 10이다.
3. 완전 성공 예산 기준은 시작 예산의 25% 이상이다.
4. 예산이 최소 유지 예산 미만이면 운영자금 고갈 실패로 처리한다.
5. 예산은 강한 장중 액션과 카드 사용의 주요 비용이다.

### 9.6 장중 지표 규칙

1. 보유 비중이 높을수록 가격 추진과 방어 효율이 오르지만 감시도와 정산 리스크도 증가한다.
2. 개인 참여도가 높을수록 가격 추진은 쉬워지지만 과열, 변동성, 감시도가 증가한다.
3. 시장 유동성이 높을수록 가격 반응이 커지지만 변동성도 증가할 수 있다.
4. 감시도가 100에 도달하면 실패한다.
5. 변동성이 높으면 가격이 목표 밴드를 벗어나거나 붕괴선에 접근할 위험이 커진다.

### 9.7 Retail Swarm 규칙

1. Retail Swarm은 개인 참여도와 동기화된다.
2. 관심 군집은 가격 추진 보너스를 제공한다.
3. 과열 군집은 감시도와 변동성을 크게 올린다.
4. 패닉 군집은 가격 하락 압력을 만든다.
5. Retail Swarm은 시각 연출이면서 실제 게임 상태 변화와 연결된다.

### 9.8 문서 이벤트 규칙

1. 문서 이벤트는 장중에 조건 기반으로 발생한다.
2. MVP에서는 한 번에 하나의 문서 이벤트만 표시한다.
3. MVP에서는 문서 이벤트가 표시되는 동안 장중 시간이 일시정지된다.
4. 플레이어가 선택지를 고르면 효과를 적용하고 장중 시간을 재개한다.
5. 각 문서 이벤트는 3개 선택지를 가진다.
6. 선택지는 안정, 공격, 회피/관망 성격으로 구성한다.
7. 문서 이벤트는 예산, 감시도, 가격 압력, 개인 참여도, 시장 유동성, 변동성, 보유 비중, 사회적 비용 중 일부에 영향을 준다.

### 9.9 정산 규칙

1. Day Settlement는 6분 장중 운용 종료 후 진행한다.
2. Final Settlement는 Day 5의 Day Settlement 이후 진행한다.
3. 실제 수익과 감시등급은 Day/Final 결과의 핵심 축이다.
4. 가격 목표만 달성해도 감시등급이 높으면 완전 성공으로 처리하지 않는다.
5. 감시등급이 낮아도 실제 수익이 낮으면 조용한 실패 또는 부분적 결과로 처리한다.
6. 높은 보유 비중은 장중 효율을 높일 수 있지만 마감 정산 리스크와 사회적 비용을 증가시킨다.
7. Day 결과는 다음 Day의 시작 조건에 영향을 줄 수 있으나, 정확한 이월 규칙은 SRS에서 확정한다.

## 10. 정산과 결과 조건

MVP의 성공 평가는 단순 목표 가격 달성이 아니라 **실제 수익**과 **감시등급**을 함께 본다.

가격 목표 달성, 예산 보전, 보유 비중 정리, 감시등급, 사회적 비용이 Day Settlement와 Final Settlement에 반영된다.

### 10.1 즉시 Run 실패

다음 중 하나에 도달하면 해당 Day 정산을 기다리지 않고 Run 전체가 실패한다.

1. 감시도 100 도달
2. 예산 최소 유지선 미만
3. 치명적 가격 붕괴선 이탈

보유 비중 0, 개인 참여도 100, 변동성 100은 MVP에서 직접 실패 조건이 아니라 패널티와 위험 증폭 요소로 사용한다.

### 10.2 감시등급

감시도는 장중에는 0~100 숫자로 표시하고, 정산에서는 감시등급으로 변환한다.

| 감시도 | 감시등급 | 의미 | 결과 영향 |
| ---: | --- | --- | --- |
| 0~24 | A — 정상 | 거의 의심 없음 | 완전 성공 가능 |
| 25~49 | B — 관찰 | 약간의 이상 흐름 | 안정 성공 가능 |
| 50~74 | C — 주의 | 감시 대상 가능성 | 위험 성공 이하 |
| 75~94 | D — 위험 | 강한 경고 상태 | 고위험 성공 또는 위험 운용 |
| 95~99 | E — 임박 | 다음 실수 시 강제 실패급 | 매우 큰 패널티 |
| 100 | F — 적발 | 즉시 실패 | Run 실패 |

### 10.3 실제 수익

MVP의 실제 수익은 복잡한 회계가 아니라 게임용 추상 점수다.

실제 수익은 다음 요소를 반영한다.

1. 대상 종목의 가격 성과
2. 플레이어의 보유 비중
3. 포지션 정리 여부
4. 잔여 예산
5. 미정리 보유 비중 패널티
6. 과열/붕괴 패널티

정확한 수익 계산식과 가중치는 SRS/SDD에서 확정한다.

### 10.4 Day Result Categories

Day Settlement는 실제 수익과 감시등급을 중심으로 결과를 분류한다.

가격 목표를 달성했더라도 감시등급이 높으면 완전 성공으로 보지 않는다.  
반대로 감시등급이 낮더라도 실제 수익이 부족하면 조용한 실패 또는 안정 운용으로 처리한다.

MVP Day 결과는 다음 8개다.

| Day 결과 | 의미 |
| --- | --- |
| 완전 성공 | 수익이 높고 감시등급도 낮음 |
| 위험 성공 | 수익은 높지만 감시등급이 중간 |
| 고위험 성공 | 수익은 높지만 감시등급이 높음 |
| 안정 운용 | 수익은 보통이고 감시등급이 낮음 |
| 위험 운용 | 수익은 보통이지만 감시등급이 높음 |
| 조용한 실패 | 감시등급은 낮지만 수익이 부족함 |
| 손실 마감 | 실제 수익이 음수 |
| 강제 실패 | 예산 고갈, 감시등급 F, 치명적 가격 붕괴 |

MVP Day 결과 판정 매트릭스는 다음 방향을 따른다.

| 실제 수익 | 감시등급 A~B | 감시등급 C | 감시등급 D~E | 감시등급 F |
| --- | --- | --- | --- | --- |
| 높음 | 완전 성공 | 위험 성공 | 고위험 성공 | 강제 실패 |
| 보통 | 안정 운용 | 위험 운용 | 위험 운용 | 강제 실패 |
| 낮음 | 조용한 실패 | 조용한 실패 | 위험 운용 | 강제 실패 |
| 손실 | 손실 마감 | 손실 마감 | 손실 마감 | 강제 실패 |

수익 구간의 정확한 수치 기준은 SRS/밸런싱 단계에서 정의한다.

### 10.5 Final Result Grades

Final Settlement는 5일 Run의 누적 실제 수익, 최종 감시등급, 평균 감시등급, 목표 달성 Day 수, 최종 예산, 최종 보유 비중, 사회적 비용을 기준으로 최종 등급을 산정한다.

MVP Final 등급은 다음 6개다.

| 최종 등급 | 의미 |
| --- | --- |
| S — 조용한 대성공 | 누적 수익 높음, 평균 감시 낮음, 강제 실패 없음, 보유 비중 안정 |
| A — 성공적 운용 | 누적 수익 좋음, 감시 관리 가능 |
| B — 위험한 성공 | 누적 수익은 충분하지만 감시 부담이 큼 |
| C — 간신히 생존 | 5일 생존은 했지만 수익/감시/예산 중 하나가 부족 |
| D — 실패한 운용 | 생존했더라도 수익이 낮거나 감시 위험이 높음 |
| F — 강제 종료 | Run 중 강제 실패 발생 |

정확한 점수 계산식과 등급 컷은 SRS/밸런싱 단계에서 정의한다.

### 10.6 보유 비중과 사회적 비용

마감 시 보유 비중이 높으면 장중 영향력은 컸더라도 실제 수익 평가와 감시 리스크에 불리하게 작용한다.  
과도한 미정리 보유 비중은 “장부상 성과는 있으나 정산 리스크가 큰 상태”로 취급한다.

MVP에서는 마감 보유 비중을 4구간으로 분류한다.

| 구간 | 보유 비중 | 정산 의미 |
| --- | ---: | --- |
| 영향력 부족 | 0~10% | 가격 영향력과 수익 기여가 낮음 |
| 안정 구간 | 10~35% | 정상 정산 구간 |
| 부담 구간 | 35~55% | 수익 잠재력은 높지만 감시/사회적 비용 증가 |
| 과점 위험 | 55% 이상 | 정산 패널티가 크고 감시등급 악화 가능 |

포지션 정리를 통해 회수한 예산은 실제 수익에 더 강하게 반영된다.  
마감 시 남은 보유 비중은 평가 성과로 일부만 반영된다.  
보유 비중이 과도하면 정산 패널티를 받는다.

정확한 실제 수익 계산식, 평가 수익 반영률, 보유 비중 패널티 수치는 SRS/밸런싱 단계에서 정의한다.

사회적 비용은 개인 참여도 과열, 패닉 발생, 변동성 폭주, 감시 이벤트 누적을 나타내는 보조 정산 지표다.

MVP의 사회적 비용은 다음 사건에서 누적되는 추상 패널티다.

1. 개인 참여도 과열
2. Retail Swarm 패닉
3. 변동성 폭주
4. 감시 문서 이벤트 악화
5. 목표 밴드 이탈 상태의 장기화

사회적 비용은 실제 피해나 실제 범죄 절차를 묘사하지 않고, 풍자적 정산 패널티와 문서 경고로만 표현한다.

사회적 비용은 Final 등급을 한 단계 낮추는 보정 요소로 사용할 수 있다.  
예를 들어 누적 수익과 감시등급 기준으로 A에 해당하더라도 사회적 비용이 매우 높으면 B로 조정될 수 있다.

정확한 사회적 비용 계산식과 등급 보정 기준은 SRS/밸런싱 단계에서 정의한다.

## 11. 주요 화면 방향

### 11.1 MVP 화면 목록

MVP의 화면은 8개로 제한한다.

| 화면 | 역할 |
| --- | --- |
| 1. 메인 메뉴 | 새 Run 시작, 설정, 기록 보기 진입 |
| 2. Run 시작 / 종목 선택 | 5일 Run 시작, 대상 섹터/종목 선택 또는 배정 |
| 3. 아침 뉴스 / 시장 브리핑 | 오늘의 뉴스, 대상 종목 프로필, 목표 조건 확인 |
| 4. 개장 전 카드 선택 | 시장 관찰, 사전 포지션 구축, 방어 자금 배정, 관망 중 선택 |
| 5. 장중 운용 화면 | 차트, 시장 보드, 예산, 감시도, 보유 비중, 수동 액션, 자동 카드 |
| 6. 문서 이벤트 팝업 | 장중 조건 기반 이벤트와 3개 선택지 |
| 7. Day 정산 화면 | 하루 수익, 감시등급, 보유 비중, 결과 분류 표시 |
| 8. Final 정산 화면 | 5일 Run 최종 등급, 누적 수익, 감시 결과, 재시작 |

문서 이벤트는 별도 전체 화면이 아니라 장중 운용 화면 위에 표시되는 모달 팝업으로 처리한다.

MVP 화면 흐름은 다음과 같다.

```text
메인 메뉴
→ Run 시작 / 종목 선택
→ Day 1 아침 뉴스 / 시장 브리핑
→ 개장 전 카드 선택
→ 개장 승인
→ 장중 운용 화면
→ 문서 이벤트 팝업
→ Day 정산
→ Day 2 아침 뉴스
→ ...
→ Day 5 정산
→ Final 정산
```

MVP에서는 도감, 카드 컬렉션, 섹터별 통계, 리플레이, 온라인 랭킹, 고급 설정 화면을 포함하지 않는다.  
이들은 P1 이후 확장 후보로 둔다.

### 11.2 MVP 제외 화면

| 화면 | 처리 |
| --- | --- |
| 상세 도감 | P1 |
| 카드 컬렉션 | P1 |
| 섹터별 통계 | P1 |
| 리플레이 | P2 |
| 설정 고급 화면 | P1 |
| 튜토리얼 전용 캠페인 | P1 |
| 온라인 랭킹 | 제외 또는 P2 |
| 전체 24개 종목 상세 시장판 | 제외 |

MVP에서는 “새 Run 시작 → 5일 진행 → 정산 → 다시 시작” 흐름을 매끄럽게 만드는 것을 우선한다.

### 11.3 Day 1 인게임 온보딩

MVP에서는 별도의 튜토리얼 모드를 제공하지 않는다.

대신 Day 1을 인게임 온보딩으로 설계한다.  
Day 1은 아침 뉴스 확인, 개장 전 카드 선택, 개장 승인, 장중 수동 액션 사용, Day 정산 확인이라는 핵심 흐름을 자연스럽게 경험하게 한다.

Day 1에서 반드시 체험시킬 핵심은 다음 4개다.

1. 아침 뉴스를 읽는다.
2. 개장 전 카드를 고른다.
3. 장중에 수동 액션을 써본다.
4. Day 정산에서 수익과 감시등급을 확인한다.

Day 1에서는 지나치게 복잡한 이벤트 조합을 피하고, 약한 시장 이벤트와 제한된 문서 이벤트를 통해 플레이어가 기본 구조를 이해하도록 한다.

| 항목 | Day 1 처리 |
| --- | --- |
| 아침 뉴스 | 가장 단순한 `섹터 호재` 또는 약한 이벤트 |
| 개장 전 카드 | 4개 모두 보여주되 짧은 설명 제공 |
| 장중 수동 액션 | 4개 모두 보이지만 툴팁 제공 |
| 문서 이벤트 | 1개만 거의 확정적으로 발생 |
| 감시/예산 실패 | 감시도 증가와 가격 붕괴 위험을 완만하게 조정 |
| 정산 화면 | “무엇을 놓쳤는지”를 짧은 힌트로 표시 |
| 재시도 유도 | Final 또는 실패 후 바로 다시 Run 시작 가능 |

플레이어가 Day 1에서 실수하거나 충분한 수익을 내지 못할 수는 있다.  
다만 MVP에서는 Day 1의 감시도 증가와 가격 붕괴 위험을 완만하게 조정해, 시스템을 이해하기 전에 즉시 실패하는 경험을 줄인다.

정산 화면은 단순 결과 표시가 아니라 다음 플레이에 도움이 되는 짧은 힌트를 제공한다.  
예를 들어 감시등급이 높으면 과열 해소나 감시 완충을, 수익이 낮으면 가격 추진이나 사전 포지션 구축을 제안한다.

이 설계의 목적은 긴 튜토리얼을 읽히는 것이 아니라, 플레이어가 한 번 플레이한 뒤 “다음에는 다르게 해보겠다”고 느끼게 만드는 것이다.

## 12. P0 우선순위

### 12.1 P0 — 1차 플레이 가능 버전

1. TypeScript + Phaser 3 + Vite 프로젝트 구성
2. 5일 Run
3. Day당 6분 장중 운용
4. 허구 섹터 8개 데이터

## 13. 리스크 및 고려사항

### 13.1 실제 금융 범죄처럼 보일 위험

대응 방향은 다음과 같다.

1. 직접 표현을 UI에서 사용하지 않는다.
2. 실제 절차, 방법, 타이밍, 회피 방식을 설명하지 않는다.
3. 모든 종목, 섹터, 뉴스는 허구로 만든다.
4. 효과는 수치 중심의 추상 게임 규칙으로 표현한다.
5. 결과 정산에는 감시와 사회적 비용을 함께 표시한다.

### 13.2 시스템 과복잡화 위험

v0.1.4에서 5일 Run, Day/Final Settlement, 콘텐츠 수량 동결이 반영되면서 범위가 커졌다.

대응 방향은 다음과 같다.

1. P0는 5일 Run만 만든다.
2. 각 Day의 장중 운용은 6분으로 제한한다.
3. 뉴스는 Day당 1개만 적용한다.
4. 개장 전 카드는 최대 1장만 사용한다.
5. 대상 종목만 상세 계산한다.
6. 비플레이어 종목은 간략 등락률만 계산한다.
7. Market Board에는 8개 종목만 표시한다.
8. Retail Swarm은 아이콘/토큰 기반으로 제한한다.

### 13.3 UI 가독성 위험

장중 화면에 너무 많은 지표가 표시될 수 있다.

대응 방향은 다음과 같다.

1. 핵심 지표는 우측 패널에 고정한다.
2. 상태는 색상과 아이콘으로 빠르게 구분한다.
3. 차트와 Retail Swarm이 서로 가리지 않게 배치한다.
4. P0에서는 모든 보조 설명을 항상 표시하지 않고 툴팁 또는 문서 패널로 분리한다.

### 13.4 밸런싱 난이도

예산, 개인 참여도, 보유 비중, 유동성, 감시도, 변동성이 서로 연결되어 밸런싱이 어려울 수 있다.

대응 방향은 다음과 같다.

1. 모든 수치는 JSON 데이터로 분리한다.
2. P0에는 디버그 패널을 둔다.
3. 가격 공식은 단순 Tick Formula로 시작한다.
4. 시뮬레이션 로그는 개발용으로만 출력한다.
5. SRS/SDD 단계에서 기본 계수와 테스트 시나리오를 확정한다.

## 14. 개발팀에게

이 제품은 실제 투자 도구가 아니다.  
또한 실제 시장 조작 방법을 알려주는 시뮬레이터도 아니다.

1차 목표는 **개장 전 문서를 읽고, 장중 시장 압력과 군중 심리를 조절하며, 목표 가격 밴드와 예산 보전 사이에서 생존하는 블랙코미디 풍자 게임**이다.

개발 시 다음 원칙을 우선한다.

1. 실제 데이터는 사용하지 않는다.
2. 모든 종목과 뉴스는 허구로 만든다.
3. 표현은 추상화한다.
4. 카드 효과는 수치 중심으로 만든다.
5. 가격 계산은 실제 금융 모델이 아니라 게임용 Tick Formula로 만든다.
6. P0는 대상 종목만 상세 계산한다.
7. 비플레이어 종목은 시장감을 주기 위한 간략 등락률만 계산한다.
8. Retail Swarm은 개인 참여도의 시각화이자 장중 압박 장치다.
9. 예산 보전과 감시도 회피가 가격 목표만큼 중요하다.
10. 플레이어가 무조건 올리는 것보다 “목표 밴드 안착”을 판단하게 만든다.
11. P0 구현은 작게 유지하고, 데이터는 JSON으로 분리해 밸런싱 가능하게 만든다.
