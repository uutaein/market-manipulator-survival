이 문서는 호가벽이 얼마나 세워졌고, 얼마나 녹았고, 얼마가 환급됐는지를 플레이어가 즉시 읽을 수 있도록 하는 가상 depth 피드백 요구를 정의한다.

---

## 0. 변경 이력

| 버전 | 변경 내용 |
| --- | --- |
| v0.2.5 | 호가벽 소진 속도 감각을 튜닝하고, 활성화/소진/붕괴/제거/만료 이벤트를 장중 UI의 짧은 로그로 표시한다. |

---

## 1. 제품 의도

v0.2.3은 호가벽 잔량을 만들었고 v0.2.4는 막대와 SIZE 변화가 부드럽게 보이도록 만들었다. 다음 단계에서는 플레이어가 “무슨 일이 일어났는지”를 숫자와 짧은 문장으로 읽을 수 있어야 한다.

이 기능의 목적은 다음이다.

1. 벽을 세운 순간 예약 예산과 생성된 가상 depth를 알려준다.
2. 벽이 녹을 때 소진된 가상 depth와 남은 환급 예산을 짧게 알려준다.
3. 벽이 완전히 붕괴하거나 제거/만료될 때 결과를 알려준다.
4. 벽 소진 속도는 강한 반대 압력에서 읽히게 녹되, 일반 tick에서 과도하게 빨리 사라지지 않도록 조정한다.

---

## 2. 범위

### 2.1 포함

| 항목 | 설명 |
| --- | --- |
| 소진 이벤트 모델 | 벽 활성화, 소진, 붕괴, 제거, 만료를 최근 이벤트로 기록 |
| 짧은 UI 로그 | 장중 화면에 최근 3개 호가벽 이벤트를 한두 줄로 표시 |
| 튜닝 값 조정 | 반대 압력 기반 소진 속도와 접촉 배율을 플레이 감각에 맞게 조정 |
| 로그 제한 | 이벤트 로그는 최근 항목만 보관해 저장/렌더 비용을 제한 |

### 2.2 제외

| 항목 | 제외 이유 |
| --- | --- |
| 실제 체결 로그 | 안전 원칙상 실제 주문/거래소/티커/계좌와 연결하지 않음 |
| 상세 주문 원장 | 이번 단계는 플레이어 피드백용 요약 로그에 한정 |
| 새 조작 버튼 | 기존 hover-row 조작 유지 |
| 가격 공식 변경 | 벽 소진과 표시 피드백만 다룸 |

---

## 3. 플레이어 경험

1. 벽을 세우면 `매수벽 형성 +160 depth / 예약 10B`처럼 요약이 보인다.
2. 벽이 녹으면 `매수벽 소진 -48 depth / 환급 7.0B`처럼 남은 상태를 읽을 수 있다.
3. 벽이 모두 녹으면 `매수벽 붕괴 / 방어선 해제`처럼 장벽 해제를 알려준다.
4. 벽을 직접 빼면 환급액이 표시된다.
5. 로그는 짧게 유지되어 기존 장중 패널을 압박하지 않는다.

---

## 4. 안전 표현 원칙

1. 로그는 실제 주문/체결이 아니라 가상 depth 이벤트로 표현한다.
2. 실제 거래소명, 실제 종목, 실제 주문 방식, 실제 시장 조작 절차를 표시하지 않는다.
3. 용어는 `가상 depth`, `예약`, `환급`, `방어선`, `압력` 같은 게임 추상 용어를 사용한다.

---

## 5. SRS 위임 사항

SRS는 다음을 확정한다.

1. 이벤트 종류와 필드
2. 이벤트 생성 조건
3. 이벤트 보관 한도
4. UI 표시 형식
5. 튜닝 값 변경 범위
6. 회귀 검증 기준
