# Market Manipulator Survival — MVP Freeze Candidate

이 문서는 `Market Manipulator Survival`의 MVP 범위를 Freeze하기 위한 승인 후보 문서다.  
기준 문서는 PRD v0.1.5이며, ADR-0001~ADR-0029의 승인 결정을 반영한다.

MVP Freeze 후보는 기능 범위, 콘텐츠 수량, 게임 루프, 화면 범위, 정산 방향, 포함/제외 범위를 고정한다.  
가격 공식, 카드 수치, 이벤트 조건, 저장 스키마, 화면 상태 전이는 SRS/밸런싱 단계에서 정의한다.

---

## 1. Product Scope

제품은 가상의 시장을 배경으로 한 싱글플레이어 풍자 게임이다.

플레이어는 실제 금융 행위를 수행하지 않는다.  
모든 종목, 섹터, 뉴스, 시장 조건은 허구이며, 게임은 실제 금융 범죄 절차를 설명하거나 재현하지 않는다.

MVP 구현 방향은 다음과 같다.

1. TypeScript + Phaser 3 + Vite 기반 경량 브라우저 게임
2. 데스크톱 브라우저 우선
3. 싱글플레이어 오프라인
4. localStorage 기반 최소 저장
5. Phaser Scene Shell + 브라우저 차트/테이블 오버레이

---

## 2. MVP Core Loop

MVP의 기본 단위는 5-Day Run이다.

각 Day는 다음 순서로 진행한다.

1. Pre-open Card selection
2. Morning News
3. Market Briefing
4. Opening Approval / stamp action
5. Intraday operation
6. Day Settlement

Day 5 이후 Final Settlement를 진행한다.

Intraday operation 기본 시간은 Day당 3분이다.
Pre-open 단계에는 시간 제한을 두지 않는다.

즉시 Run 실패 조건은 다음이다.

1. 예산 고갈
2. 감시도 100 도달
3. 치명적 가격 붕괴

MVP에는 조기 성공 엔딩을 넣지 않는다.

## 4. Fictional Market Scope

MVP 섹터는 8개다.

1. 식량·농업
2. 에너지·전력망
3. 바이오·임상
4. 자동화·AI
5. 칩·장비
6. 결제·핀테크
7. 미디어·게임
8. 밈·테마

각 섹터에는 3개 허구 종목이 있으며, 총 24개 종목을 사용한다.

플레이어는 Run 시작 시 섹터와 종목을 직접 선택한다.  
모든 섹터와 종목은 처음부터 선택 가능하며, 해금 구조는 제공하지 않는다.

종목명과 섹터는 고정하지만, 세부 Asset Profile은 Run 시작 시 섹터별 3개 종목에 안정/표준/고위험 성향을 하나씩 무작위 배정해 결정한다.  
이 성향은 플레이어에게 직접 표시하지 않는다.

최종 종목 성향은 다음 요소의 조합으로 결정된다.

```text
섹터 기본 성향
+ Run 시작 시 랜덤 배정된 Asset 성향
+ Today Condition
+ Morning News
+ Market Aftereffect
```

## 5. Morning News and Pre-open

MVP Morning News는 5개 템플릿으로 제한한다.

1. 섹터 호재
2. 섹터 악재
3. 시장 침체
4. 규제 경고
5. 과열 확산

각 Day에는 3개의 Morning News가 표시된다.
구성은 섹터 뉴스 1개와 허구 종목 뉴스 2개다. 비플레이어 종목 뉴스는 시장 보드 맥락과 배지에는 표시하지만, 플레이어 종목 가격 계산에는 직접 반영하지 않는다.

MVP Pre-open Card는 4개다.

1. 사전 포지션 확보
2. 뉴스 배정
3. 종목 분석
4. 관망

플레이어는 Morning News 공개 전에 개장 전 카드 중 최대 1장을 선택할 수 있다.
카드를 사용하지 않는 선택은 `관망`으로 표현한다.

`뉴스 배정`은 하나의 MVP 카드지만 `호재` 또는 `악재` 방향을 고를 수 있고, 대상은 플레이어가 선택한 허구 종목으로 고정한다.

카드 선택 후 Morning News와 Market Briefing을 확인하고, `개장 승인` 버튼 또는 도장 액션을 통해 장중 단계로 진입한다.

## 6. Intraday Scope

MVP 장중 핵심 지표는 다음이다.

1. Budget
2. Price
3. Holding Ratio / Market Influence
4. Personal Participation
5. Market Liquidity
6. Surveillance
7. Volatility
8. Market Pressure

MVP 수동 액션은 4개다.

1. 유동성 공급
2. 매수봇
3. 매도봇
4. 포지션 정리

MVP 자동 카드는 8개다.

1. 관심 신호
2. 유동성 순환
3. 가격 지지
4. 변동성 흡수
5. 뉴스 증폭
6. 감시 완충
7. 경쟁 견제
8. 정리 루틴

자동 카드는 Lv.1~Lv.3까지만 성장한다.  
카드 진화, 카드 조합 시너지, 희귀/전설 카드는 MVP에서 제외한다.

## 7. Document Events

MVP 문서 이벤트는 8개다.

1. 이상 흐름 질의서
2. 시장 과열 경보
3. 유동성 경색 보고
4. 커뮤니티 폭주 알림
5. 경쟁 데스크 개입 보고
6. 급락 위험 통지
7. 내부 리스크 메모
8. 마감 전 정리 요청

문서 이벤트는 장중에 조건 기반으로 발생한다.  
MVP에서는 이벤트가 표시되는 동안 장중 시간이 일시정지된다.

각 문서 이벤트는 3개 선택지를 가진다.  
선택지는 안정, 공격, 회피/관망 성격으로 구성한다.

## 8. Market Board and Simulation Scope

MVP 시장 보드는 장중 시장이 살아 있다는 감각을 주기 위해 세 영역으로 나눈다.

1. 플레이어 대상 종목 1개: 상세 계산
2. 같은 섹터 경쟁 종목 2개: 간략 계산
3. 타 섹터 평균 7개: 간략 계산
4. 마켓 대시보드: 24개 개별 종목 거래대금 순위 중 내 종목 주변 구간

24개 전체 종목의 실시간 상세 시뮬레이션은 MVP에서 제외한다.

뉴스 영향을 받는 섹터나 종목은 시장 보드에서 배지 또는 간단한 상태 표시로 보여준다.

마켓 대시보드의 개별 종목 행에는 거래대금 막대를 표시하지 않고, 순위/등락률/거래량/거래대금 숫자와 간단한 변화 표시만 사용한다.

## 9. Settlement Scope

Day Settlement는 실제 수익과 감시등급을 중심으로 결과를 분류한다.

MVP Day 결과는 8개다.

1. 완전 성공
2. 위험 성공
3. 고위험 성공
4. 안정 운용
5. 위험 운용
6. 조용한 실패
7. 손실 마감
8. 강제 실패

Final Settlement는 5일 Run의 누적 실제 수익, 최종 감시등급, 평균 감시등급, 목표 달성 Day 수, 최종 예산, 최종 보유 비중, 사회적 비용을 기준으로 최종 등급을 산정한다.

MVP Final 등급은 6개다.

1. S — 조용한 대성공
2. A — 성공적 운용
3. B — 위험한 성공
4. C — 간신히 생존
5. D — 실패한 운용
6. F — 강제 종료

정확한 수익 구간, 점수 계산식, 등급 컷은 SRS/밸런싱 단계에서 정의한다.

## 10. Carryover, Seed, and Restart

MVP의 5일 Run에서는 일부 상태가 Day 간에 이월된다.

| 항목 | 처리 |
| --- | --- |
| 예산 | 이월 |
| 누적 실제 수익 | 이월 |
| 보유 비중 | 이월 |
| 감시도 | 일부 이월 |
| 사회적 비용 | 누적 |
| 자동 카드 레벨 | 이월 |
| 개인 참여도 | 완전 초기화하지 않고 감소 이월 |
| 뉴스 효과 | 일부 잔류 효과 이월 |
| 시장 유동성 | 대부분 초기화 |
| 변동성 | 대부분 초기화하되 과열/패닉 여파는 일부 가능 |
| 개장 전 카드 효과 | 해당 Day만 적용 |

각 Run은 내부 Run Seed를 가진다.  
Run Seed는 Run별 랜덤 Asset Profile, Morning News 대상, Today Condition, 일부 시장 여파 초기 조건을 재현하기 위한 기준값이다.

Run 실패 또는 Final Settlement 이후에는 다음 선택지를 제공한다.

1. 같은 조건으로 재시작
2. 새 Run 시작

Seed 직접 입력, Seed 공유, 일일 챌린지는 MVP에서 제외한다.

## 11. Screen Scope

MVP 화면은 8개다.

1. 메인 메뉴
2. Run 시작 / 종목 선택
3. 개장 전 카드 선택
4. 아침 뉴스 / 시장 브리핑
5. 장중 운용 화면
6. 문서 이벤트 팝업
7. Day 정산 화면
8. Final 정산 화면

문서 이벤트는 별도 전체 화면이 아니라 장중 운용 화면 위에 표시되는 모달 팝업으로 처리한다.

별도 튜토리얼 모드는 제공하지 않는다.  
Day 1을 인게임 온보딩으로 설계한다.

## 12. Storage Scope

MVP에서는 로컬 저장만 사용한다.

저장 대상은 다음으로 제한한다.

1. 진행 중 Run 상태
2. 최근 Final Settlement 결과
3. 최고 Final 등급
4. 최고 누적 수익

상세 플레이 로그, 리플레이 데이터, 클라우드 저장, 온라인 랭킹은 MVP에서 제외한다.

## 13. Explicitly Deferred

MVP에서 제외하는 주요 기능은 다음과 같다.

1. 카드 조합 시너지, 카드 진화, 희귀/전설 카드
2. 24개 전체 종목의 실시간 상세 시뮬레이션
3. 전체 시장 히트맵과 섹터별 상세 통계
4. 24개 이상의 수작업 뉴스 콘텐츠
5. 장중 속보 뉴스, 장중 복수 속보 동시 발생, 뉴스 체인
6. 정교한 실현/평가손익 회계 모델
7. 세금, 수수료, 슬리피지, 정교한 호가창 단위 체결 계산
8. 별도 튜토리얼 모드, 스토리 캠페인, 단계별 미션
9. 카드/종목 도감, 고급 통계, 플레이 로그 상세, 리플레이
10. 온라인 랭킹, 클라우드 저장, 멀티플레이
11. Seed 직접 입력, Seed 공유, 일일 챌린지
12. Electron 패키징, 모바일/iPad 대응
13. 고해상도 캐릭터 일러스트, 복잡한 애니메이션, 섹터별 전용 아트, 컷신

새로운 아이디어는 MVP 범위에 추가하지 않고 P1/P2 후보로 기록한다.
