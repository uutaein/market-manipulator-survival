## 0. 변경 이력

| 버전 | 변경 내용 |
| --- | --- |
| v0.2.1 | 기존 장중 호가창/매물대 패널의 ASK/BID 행에 `매도벽 세우기` / `매수벽 세우기` 호버 액션을 추가. 별도 수동 액션 버튼은 추가하지 않고, 허구 주문장 깊이와 가격 반응성에만 영향을 주는 안전한 게임 추상화로 정의 |

## 1. 제품 의도

장중 화면의 호가창은 현재 sell-side와 buy-side 깊이를 읽는 정보 패널이다.

v0.2.1에서는 이 패널을 아주 작은 전술 입력 지점으로 확장한다. 플레이어는 ASK 3줄 또는 BID 3줄 위에 마우스를 올려 해당 방향의 벽 세우기 액션을 확인하고, 줄 자체를 눌러 실행한다.

이 기능의 목적은 다음이다.

1. 호가창을 단순 장식이 아니라 가격 반응성 판단 도구로 만든다.
2. 별도 버튼을 늘리지 않고, 정보가 있는 위치에서 바로 행동하게 한다.
3. `매수봇` / `매도봇`보다 작은 단기 압력 조절 선택지를 만든다.
4. 의뢰모드의 상단 제한, 하단 방어, 밴드 유지 목표에서 더 섬세한 장중 대응을 제공한다.

---

## 2. 범위

### 2.1 포함

| 항목 | 설명 |
| --- | --- |
| `매수벽 세우기` | BID 행에서 실행하는 단기 buy-side depth 보강 액션 |
| `매도벽 세우기` | ASK 행에서 실행하는 단기 sell-side depth 보강 액션 |
| 행 호버 표시 | ASK/BID 각 3줄에 마우스를 올리면 실행 가능한 액션 라벨 표시 |
| 단기 효과 | 클릭한 호가 줄의 허구 호가 깊이, 해당 가격 레벨의 임시 방어선/저항선, 가격 반응성, 소폭의 시장 압력/관심/감시 수치 변화 |
| 예산 예약/환급 | 늘어나는 벽 볼륨에 비례해 예산을 예약하고, 벽 제거 또는 만료 시 환급 |
| 공유 사용 | 자유모드와 의뢰모드 모두 같은 장중 엔진에서 사용 |

### 2.2 제외

| 항목 | 제외 이유 |
| --- | --- |
| 별도 큰 버튼 2개 | 장중 버튼 수를 늘리지 않기 위해 제외 |
| 실제 주문 가격/수량 입력 | 실제 금융 절차처럼 보일 수 있어 제외 |
| 정교한 호가별 체결 계산 | v0.2.1 목표보다 과한 시뮬레이션 |
| Day 간 벽 효과 이월 | 단기 장중 전술로만 유지 |
| 실제 시장 데이터 연결 | 제품 안전 원칙상 제외 |

## 3. 플레이어 경험

호가창은 다음처럼 작동해야 한다.

1. ASK 행에 호버하면 행 위에 `매도벽 세우기`가 보인다.
2. BID 행에 호버하면 행 위에 `매수벽 세우기`가 보인다.
3. 실행 가능한 상태면 클릭으로 즉시 해당 행의 벽 효과를 시작하고 SIZE가 크게 증가한다.
4. 실행 불가 상태면 행은 비활성 이유를 간단히 보여준다.
5. 활성 중인 벽은 클릭한 가격 줄의 SIZE와 깊이를 즉시 눈에 띄게 두껍게 보여준다.
6. 매수벽은 녹거나 제거되기 전까지 해당 가격 레벨 아래로 가격이 내려가는 것을 막는다.
7. 매도벽은 녹거나 제거되기 전까지 해당 가격 레벨 위로 가격이 올라가는 것을 막는다.
8. 활성 중인 벽을 다시 클릭하면 벽을 제거하고 예약 예산을 환급한다.
9. 다른 ASK/BID 줄은 독립적으로 벽을 세울 수 있다.
10. 같은 줄이 쿨다운 중이면 그 줄만 새로 세울 수 없다.

이 기능은 메인 수동 액션을 대체하지 않는다.  
`매수봇`과 `매도봇`은 여전히 보유 비중과 시장 압력을 크게 바꾸는 주요 액션이고, 호가벽은 가격 반응성을 짧게 기울이는 보조 입력이다.

## 4. 모드별 의미

| 모드 | 의미 |
| --- | --- |
| 자유모드 | 목표 밴드 접근, 급락 완충, 과열 제한을 위한 짧은 장중 보조 조작 |
| 의뢰모드 | 터치 후 유지, 하단 방어, 상단 제한, 밴드 유지 조건에서 섬세한 압력 보정 |

의뢰모드에서 벽 사용은 계약 순성과에 직접 보너스를 주지 않는다. 사용 비용과 감시 리스크가 정산에 반영되므로, 적게 쓰고 조건을 맞출수록 효율이 좋다.

---

## 5. 안전 표현 원칙

이 기능은 실제 주문 행위나 실제 시장 조작 절차를 설명하지 않는다.

1. 모든 수치는 허구 게임 수치다.
2. 호가창은 실제 시장 데이터가 아니라 seeded fictional depth다.
3. 플레이어는 실제 주문 가격이나 수량을 입력하지 않는다.
4. UI는 `호가 깊이`, `BID`, `ASK`, `시장 압력`, `감시도`, `리스크` 같은 추상 표현을 사용한다.
5. 설명문은 실제 금융 범죄, 실제 주문 전술, 회피 방식, 실행 타이밍을 다루지 않는다.

---

## 6. SRS 위임 사항

SRS는 다음을 확정한다.

1. 예산 예약량, 지속 시간, 쿨다운, 환급 규칙
2. 호가 깊이 보강량
3. 가격 반응성에 미치는 방향별 영향
4. 호가 레벨별 활성/쿨다운 상태
5. 보유 0, 예산 부족, 일시정지, 쿨다운 상태의 입력 제한
6. 호버/클릭/비활성 표시 요구사항
7. 회귀 테스트 수용 기준
