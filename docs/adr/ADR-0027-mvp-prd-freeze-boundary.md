# ADR-0027 — MVP PRD Freeze Boundary

| 항목 | 내용 |
| --- | --- |
| 상태 | Accepted |
| 날짜 | 2026-06-13 |
| 관련 문서 | PRD v0.1.5, ADR-0020, ADR-0025, ADR-0026 |

## Context

PRD v0.1.5에서 MVP의 주요 기능 범위, 콘텐츠 수량, 게임 루프, 화면 범위, 정산 방향, 포함/제외 범위가 대부분 확정되었다.

이후에도 PRD에 새 기능을 계속 추가하면 MVP 범위가 다시 커지고, SRS/밸런싱 단계로 넘어가기 어렵다.

## Decision

MVP PRD는 기능 범위, 콘텐츠 수량, 게임 루프, 화면 범위, 정산 방향, 포함/제외 범위를 확정하는 문서로 둔다.

가격 Tick 공식, 카드 비용/쿨다운/효과량, 문서 이벤트 발생 조건, 뉴스 수치, 시장 여파 잔류율, 감시도 계산식, 실제 수익 계산식, 저장 스키마, 화면별 상태 전이는 SRS/밸런싱 단계에서 정의한다.

MVP Freeze 이후에는 새 기능을 PRD MVP 범위에 추가하지 않는다.  
새로운 아이디어는 P1/P2 후보로 기록한다.

핵심 루프를 바꾸는 변경은 별도 승인 없이는 반영하지 않는다.

## Consequences

PRD v0.1.5는 MVP 범위 동결 기준 문서가 된다.

추가 아이디어는 MVP를 다시 넓히지 않고 P1/P2 후보로 관리한다.

SRS/밸런싱 단계는 확정된 범위 안에서 수치, 공식, 조건, 화면 상태 전이를 구체화한다.

## Alternatives Considered

### Continue PRD Feature Expansion

기각한다.  
기능과 콘텐츠가 계속 추가되면 MVP가 작게 유지되지 않고, SRS로 넘어갈 기준이 흐려진다.

### Move Formulas into PRD

기각한다.  
PRD는 제품 범위와 의사결정 문서로 유지하고, 공식과 수치 튜닝은 SRS/밸런싱 단계에서 다룬다.
