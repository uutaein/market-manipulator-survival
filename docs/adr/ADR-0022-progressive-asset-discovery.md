# ADR-0022 — Progressive Asset Discovery

| 항목 | 내용 |
| --- | --- |
| 상태 | Accepted |
| 날짜 | 2026-06-13 |
| 관련 문서 | PRD v0.1.5, ADR-0004, ADR-0007, ADR-0019, ADR-0021 |

## Context

MVP의 각 종목은 내부적으로 Asset Profile을 가진다.

이후 ADR-0023에서 종목명과 섹터는 고정하되, 세부 Asset Profile 성향은 Run 시작 시 섹터별로 무작위 배정하는 것으로 보정했다.

다만 Run 시작 전부터 규모, 유통 두께, 변동성, 감시 민감도, 예산 효율 같은 세부 지표를 모두 공개하면 플레이어가 몇 판 안에 종목을 안정형/고위험형으로 빠르게 분류할 수 있다.

이 경우 종목 선택이 단순한 난이도 선택으로 바뀌고, 5일 Run 동안 종목 성향을 학습하는 재미가 약해진다.

## Decision

MVP에서는 종목의 안정형, 표준형, 고위험형 내부 타입을 플레이어에게 직접 표시하지 않는다.

또한 Run 시작 전부터 규모, 유통 두께, 변동성, 감시 민감도, 예산 효율 같은 세부 지표를 모두 공개하지 않는다.

플레이어는 종목명, 섹터, 짧은 브리핑만 보고 Run을 시작한다.  
종목 성향은 장중 반응, Day 정산 메모, 시장 여파, 반복 플레이 경험을 통해 점진적으로 드러난다.

## Rationale

초기 화면에서 종목 정보를 너무 많이 제공하면 플레이어가 종목을 빠르게 분류하고, 선택이 탐색이 아니라 정답 찾기에 가까워진다.

이 게임은 5일 Run 구조를 가지고 있으므로, 종목의 진짜 성향은 날짜를 넘기면서 드러나는 편이 더 적합하다.

## Consequences

Run 시작 전 정보는 제한된다.

플레이어는 Day 1에서 종목을 완전히 이해하지 못할 수 있다. 이는 의도된 설계다.

Day 정산 메모와 시장 여파는 종목 성향을 학습시키는 핵심 장치가 된다.

정확한 힌트 노출량, 메모 문구, 지표 공개 타이밍은 SRS/UX 단계에서 정의한다.
