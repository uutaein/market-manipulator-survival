# ADR-0023 — Run Random Asset Profile

| 항목 | 내용 |
| --- | --- |
| 상태 | Accepted |
| 날짜 | 2026-06-13 |
| 관련 문서 | PRD v0.1.5, ADR-0004, ADR-0021, ADR-0022 |

## Context

ADR-0022는 종목의 내부 타입과 세부 지표를 플레이어에게 직접 공개하지 않고, 장중 반응과 정산 메모를 통해 점진적으로 드러내기로 결정했다.

다만 종목별 성향이 항상 고정되어 있으면 반복 플레이를 통해 특정 종목이 안정형인지 고위험형인지 빠르게 외워질 수 있다. 이 경우 Progressive Asset Discovery의 효과가 약해지고, 종목 선택이 장기적으로 정답 암기에 가까워진다.

## Decision

MVP에서는 종목명과 섹터는 고정하지만, 각 종목의 세부 성향은 Run 시작 시 무작위로 배정한다.

각 섹터에는 3개의 허구 종목이 존재한다.  
Run 시작 시 해당 섹터의 3개 종목에는 안정 성향, 표준 성향, 고위험 성향이 각각 하나씩 무작위로 배정된다.

이 성향은 플레이어에게 직접 표시하지 않는다.  
플레이어는 Run 시작 시 종목명, 섹터, 짧은 브리핑만 확인할 수 있으며, 실제 성향은 장중 반응, Day 정산 메모, 시장 여파를 통해 점진적으로 유추한다.

최종 종목 성향은 다음 요소의 조합으로 결정된다.

```text
섹터 기본 성향
+ Run 시작 시 랜덤 배정된 Asset 성향
+ Today Condition
+ Morning News
+ Market Aftereffect
```

## Rationale

섹터별 3종목에 안정/표준/고위험 성향을 하나씩 배정하면, Run마다 종목 성향이 달라져 반복 플레이의 탐색성이 유지된다.

동시에 완전 무작위 수치 생성이 아니므로 MVP 밸런싱 범위를 통제할 수 있다. 섹터마다 세 성향이 하나씩 존재하기 때문에 특정 섹터가 한 Run에서 지나치게 쉽거나 어렵게 몰리는 문제도 줄어든다.

## Consequences

종목명은 플레이어가 기억할 수 있는 고정 콘텐츠로 남지만, 같은 종목이라도 Run마다 세부 반응이 달라질 수 있다.

플레이어는 Run 시작 전 정보만으로 정답을 고르기 어렵고, 장중 반응과 정산 메모를 통해 성향을 읽어야 한다.

Day 정산 메모, 시장 여파, 반복 플레이 힌트의 중요도가 커진다.

정확한 성향별 수치 범위, 섹터 기본 성향, 힌트 문구, 무작위 배정 시드 규칙은 SRS/밸런싱 단계에서 정의한다.

## Alternatives Considered

### Fixed Asset Profile per Asset

기각한다.  
반복 플레이에서 종목 성향이 빠르게 암기되어 종목 선택의 탐색성이 줄어든다.

### Fully Random Asset Profile

기각한다.  
MVP 밸런싱 범위가 커지고, 섹터별 정체성과 5일 Run의 예측 가능성이 약해진다.
