## 1. Context

섹터와 종목 수량이 확정되면서, 각 종목이 이름만 다른 대상이 되지 않도록 기본 성향을 정의할 필요가 생겼다.

사용자는 종목마다 랜덤으로 결정된 규모나 특성이 있어야 한다고 제안했다.  
다만 완전 랜덤이면 플레이어가 종목별 성향을 학습하기 어렵고, 완전 고정이면 반복 플레이 변동성이 줄어든다.

P0의 목적은 종목별 정체성과 라운드별 변화를 동시에 확보하는 것이다.

---

## 2. Decision

### 2.1 모든 종목에 고정 Asset Profile을 부여한다

P0에서는 모든 허구 종목에 고정 Asset Profile을 둔다.

Asset Profile은 다음 값을 포함한다.

| 값 | 의미 |
| --- | --- |
| 규모 | 소형, 중형, 대형 |
| 유통 두께 | 얇음, 보통, 두꺼움 |
| 기본 관심도 | 시작 개인 참여도와 뉴스 반응성 |
| 기본 변동성 | 가격 흔들림 정도 |
| 감시 민감도 | 감시도 증가 보정 |
| 경쟁 강도 | 경쟁 압박 등장 가능성 |
| 예산 효율 | 예산 1당 시장 압력 효율 |
| 목표 난이도 | 목표 밴드 산정 난이도 |

### 2.2 규모는 3단계로 단순화한다

| 규모 | 특징 |
| --- | --- |
| 소형 | 적은 예산으로 잘 움직이지만 변동성·감시 민감도가 높음 |
| 중형 | 표준형 |
| 대형 | 움직이기 어렵지만 안정적이고 감시 민감도가 낮음 |

### 2.3 유통 두께는 3단계로 단순화한다

| 유통 두께 | 특징 |
| --- | --- |
| 얇음 | 가격 반응이 크지만 급락 위험이 높음 |
| 보통 | 표준형 |
| 두꺼움 | 많은 예산이 필요하지만 안정적 |

### 2.4 라운드마다 Today Condition을 적용한다

종목 프로필은 기본적으로 고정한다.  
라운드 시작 시 Today Condition을 통해 일부 수치만 ±10~15% 범위에서 변동한다.

변동 대상 예시는 다음과 같다.

1. 관심도
2. 변동성
3. 예산 효율
4. 감시 민감도

완전 랜덤 프로필은 사용하지 않는다.

### 2.5 목표 밴드와 시작 예산은 프로필에서 산정한다

P0에서는 시작 예산 기본값 100을 유지하되, 규모와 목표 난이도에 따라 보정할 수 있다.

예시는 다음과 같다.

| 규모 | 시작 예산 방향 | 목표 상승 밴드 방향 |
| --- | ---: | --- |
| 소형 | 낮음 | 넓고 높음 |
| 중형 | 표준 | 표준 |
| 대형 | 높음 | 낮고 좁음 |

정확한 수치는 SRS/SDD에서 확정한다.

---

## 3. Consequences

### 3.1 Positive Consequences

1. 종목마다 플레이 감각이 달라진다.
2. 플레이어가 종목 성향을 학습할 수 있다.
3. Today Condition으로 매 라운드 변동성을 확보한다.
4. 완전 랜덤보다 밸런싱이 쉽다.
5. 시작 예산과 목표 밴드를 종목 성향과 연결할 수 있다.

### 3.2 Negative Consequences

1. 24개 종목 프로필 데이터를 작성해야 한다.
2. 각 프로필의 밸런싱 비용이 생긴다.
3. Today Condition이 너무 약하면 변화가 안 느껴질 수 있다.
4. Today Condition이 너무 강하면 고정 프로필의 의미가 약해질 수 있다.

---

## 4. Alternatives Considered

### 4.1 매 라운드 완전 랜덤 프로필 생성

반복성은 낮아지지만 플레이어 학습이 어렵다.  
P0에는 적합하지 않아 채택하지 않는다.

### 4.2 모든 종목 동일 스탯 사용

구현은 쉽지만 종목 선택과 시장 보드의 의미가 약해진다.  
채택하지 않는다.

### 4.3 섹터만 다르고 종목 프로필은 없음

섹터 이벤트는 살아나지만 개별 종목의 정체성이 부족하다.  
채택하지 않는다.

---

## 5. Follow-ups

1. 24개 종목의 실제 Asset Profile 값은 SRS/SDD에서 작성한다.
2. Today Condition의 랜덤 범위는 우선 ±10~15%로 둔다.
3. 목표 밴드 공식과 시작 예산 보정 공식은 SRS/SDD에서 확정한다.
4. 개장 전 화면에는 Asset Profile과 Today Condition 요약을 표시한다.
