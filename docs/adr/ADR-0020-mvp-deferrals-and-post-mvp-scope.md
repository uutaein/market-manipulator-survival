# ADR-0020 — MVP Deferrals and Post-MVP Scope

| 항목 | 내용 |
| --- | --- |
| 문서 종류 | ADR (Architecture Decision Record) |
| 제품명 | Market Manipulator Survival |
| 상태 | Accepted |
| 날짜 | 2026-06-13 |
| 관련 문서 | PRD v0.1.4, ADR-0011, ADR-0013, ADR-0016, ADR-0018, ADR-0019 |
| 결정 범위 | MVP 제외 기능, P1/P2 후속 후보, MVP 복잡도 제한 |

---

## Context

MVP의 목적은 다음 핵심 루프가 재미있는지 검증하는 것이다.

```text
5일 Run
→ 아침 뉴스
→ 개장 전 선택
→ 장중 압박
→ 문서 이벤트
→ 정산
```

따라서 재미 검증에 직접 필요 없거나 콘텐츠 제작량, 밸런싱 범위, 구현 복잡도를 크게 늘리는 기능은 MVP에서 제외해야 한다.

---

## Decision

MVP에서 제외하는 주요 기능은 다음과 같다.

| 제외 항목 | 후속 단계 |
| --- | --- |
| 카드 조합 시너지 | P1 |
| 카드 진화 | P1 |
| 희귀/전설 카드 | P1 |
| 카드 덱 저장 | P1/P2 |
| 24개 전체 종목 실시간 상세 시뮬레이션 | P1 |
| 전체 시장 히트맵 | P1 |
| 섹터별 상세 통계 | P1 |
| 24개 이상 수작업 뉴스 | P1 |
| 장중 속보 뉴스 | P1 |
| 복수 뉴스 동시 발생 | P1 |
| 뉴스 체인/후속 보도 | P2 |
| 정교한 실현/평가손익 분리 | P1 |
| 세금/수수료/슬리피지 | P2 |
| 호가창 단위 계산 | P2 |
| 별도 튜토리얼 모드 | P1 |
| 스토리 캠페인 | P1/P2 |
| 단계별 미션 | P1 |
| 카드/종목 도감 | P1 |
| 고급 통계/플레이 로그 상세 | P1/P2 |
| 리플레이 | P2 |
| 온라인 랭킹 | P2 |
| 클라우드 저장 | P2 |
| Electron 패키징 | P1 |
| 모바일/iPad 대응 | P2 |
| 고해상도 캐릭터 일러스트 | P2 |
| 복잡한 애니메이션 | P1/P2 |
| 섹터별 전용 아트 | P1 |
| 컷신 | P2 |

핵심 요약은 다음이다.

```text
카드 시너지, 전체 시장 상세 시뮬레이션, 많은 뉴스,
복잡한 회계, 도감/통계, 플랫폼 확장은 모두 MVP에서 제외한다.
```

---

## Rationale

MVP는 작지만 실제로 한 판이 돌아가는 게임이어야 한다.

자동 카드, 수동 액션, 개장 전 카드, 문서 이벤트, 정산 체계만으로도 이미 밸런싱 대상이 충분하다.  
후속 기능을 MVP에 섞으면 핵심 루프가 재미있는지 검증하기 전에 콘텐츠와 시스템 복잡도가 먼저 커진다.

---

## Consequences

### Positive

1. MVP 구현 범위가 작아진다.
2. 밸런싱 대상이 핵심 루프에 집중된다.
3. 문서, 뉴스, 카드 콘텐츠 제작량을 통제할 수 있다.
4. 브라우저 기반 첫 플레이 검증에 집중할 수 있다.

### Negative

1. 카드 빌드 다양성은 P1 전까지 제한된다.
2. 장기 진행, 도감 수집, 통계 분석 같은 메타 재미는 MVP에 없다.
3. 시장 전체 시뮬레이션을 기대하는 사용자에게는 범위가 작게 느껴질 수 있다.

### Mitigations

1. MVP는 5일 Run의 반복성과 정산 피드백으로 재플레이를 유도한다.
2. 자동 카드 8개와 문서 이벤트 8개로 최소 빌드/돌발성을 제공한다.
3. P1/P2 후보를 명시해 후속 확장 경로를 남긴다.

---

## PRD Impact

PRD v0.1.4는 다음을 반영한다.

1. MVP 제외 및 후속 후보 표를 추가한다.
2. P1/P2 후보를 명확히 분리한다.
3. MVP는 5일 Run 핵심 루프 검증에 집중한다고 명시한다.
