## 1. Context

PRD v0.1.1은 P0 기술 스택, 세션 길이, 저장 방식, 콘텐츠 수량, 승패 조건, 안전 표현 원칙을 확정했다.

그러나 핵심 자원 설계에는 다음 모호성이 남아 있었다.

1. 기존 `유동성` 자원이 플레이어가 직접 관리하는 핵심 자원인지, 시장 상태를 나타내는 보조 수치인지 불명확했다.
2. 플레이어가 강한 작전 카드를 무제한으로 사용하지 못하게 하는 비용 구조가 부족했다.
3. 단순히 목표 가격 밴드에 도달하면 성공하는 구조는 전략성이 약했다.
4. 플레이어가 “가격을 올리는 것”만 최적화하면 소재가 실제 주가조작처럼 보일 위험이 있었다.
5. 사용자는 “세력도 예산이 있고, 예산이 일정 이하로 떨어지면 실패하며, 예산을 보전하면서 목표를 터트리면 성공”하는 구조를 원했다.
6. 동시에 “자전거래” 같은 직접적 표현은 너무 주가조작처럼 느껴질 수 있으므로 단순하고 추상적인 게임 표현이 필요했다.

P0의 목적은 실제 금융 행위 재현이 아니라, 예산을 태워 시장 압력을 만들고 그 비용을 감당하는 블랙코미디적 자원 관리 루프를 검증하는 것이다.

---

## 2. Decision

다음 결정을 P0 기준으로 채택한다.

### 2.1 기존 유동성 자원은 예산으로 통합한다

P0에서는 `유동성`을 독립 자원으로 유지하지 않는다.

대신 `예산`을 플레이어의 핵심 생존 자원으로 사용한다.

예산은 다음 역할을 가진다.

1. 작전 카드 사용 비용
2. 가격 방어 비용
3. 관심도 상승 비용
4. 감시도 완화 비용
5. 과열 해소 또는 포지션 정리를 통한 회복 대상
6. 세션 정산의 주요 평가 기준

### 2.2 P0 예산 기본값을 확정한다

P0 기본값은 다음으로 둔다.

| 항목 | 값 |
| --- | ---: |
| 시작 예산 | 100 |
| 최소 유지 예산 | 10 |
| 완전 성공 예산 기준 | 시작 예산의 25% 이상 |
| 실패명 | 운영자금 고갈 |

예산이 최소 유지 예산 미만으로 내려가면 라운드는 실패한다.

### 2.3 성공 조건을 예산 보전형으로 갱신한다

P0 결과는 다음 세 가지로 나눈다.

| 결과 | 조건 |
| --- | --- |
| 완전 성공 | 10분 생존 + 목표 가격 밴드 도달 또는 유지 + 예산 25% 이상 보전 |
| 부분 성공 | 10분 생존 + 목표 가격 밴드 도달 또는 유지 + 예산 보전 기준 미달 |
| 실패 | 감시도 100, 예산 최소 유지선 미만, 가격 붕괴선 이탈 중 하나 발생 |

이 결정은 가격만 올리는 전략보다, 목표 달성과 자원 보전의 균형을 핵심 재미로 만들기 위한 것이다.

### 2.4 “순환 유동성” 카드를 P0 핵심 카드로 추가한다

P0에는 `순환 유동성` 카드를 포함한다.

이 카드는 다음 의도를 가진다.

1. 예산을 소모한다.
2. 단기 가격 압력을 만든다.
3. 반복 사용 시 감시도와 변동성을 증가시킨다.
4. 실제 금융 범죄 절차는 설명하지 않는다.
5. 게임 수치와 풍자 문구만 제공한다.

허용되는 설명 예시는 다음과 같다.

```text
예산을 사용해 단기 가격 압력을 만든다.
효과는 강하지만 반복 사용 시 감시도와 변동성이 증가한다.
```

허용하지 않는 설명 예시는 다음과 같다.

```text
특정 거래 방식으로 거래량을 부풀려 시장 참여자가 있는 것처럼 보이게 한다.
```

### 2.5 직접적 금융 범죄 용어를 추상 표현으로 치환한다

UI, 튜토리얼, 카드 설명, PRD의 제품 지향 문구에서는 다음 직접 표현을 사용하지 않는다.

| 직접 표현 | 추상 표현 |
| --- | --- |
| 자전거래 | 순환 유동성 |
| 시세조종 | 시장 압력 조절 |
| 허수 주문 | 신호 주문 또는 관심 신호 |
| 작전 세력 | 운용 데스크 또는 작전 데스크 |
| 개미 꼬시기 | 관심도 상승 |
| 펌핑 | 가격 추진 |
| 덤핑 | 과열 해소 또는 정산 |
| 물량 떠넘기기 | 시장 흡수 또는 포지션 정리 |

문서 내부에서 위험 설명을 위해 직접 표현을 언급해야 할 경우에도, 사용자-facing 기능명으로는 사용하지 않는다.

### 2.6 점수 계산에 예산 보전율을 포함한다

점수는 가격 성과만으로 계산하지 않는다.

P0 점수는 다음 요소를 반영한다.

1. 목표 밴드 달성 점수
2. 예산 보전율 보너스
3. 감시도 잔여분 보너스
4. 변동성 안정 보너스
5. 사회적 비용 패널티
6. 실패 조건 패널티

예산 보전율은 정산 화면에서 명시적으로 표시한다.

---

## 3. Consequences

### 3.1 Positive Consequences

1. 플레이어의 핵심 선택이 명확해진다.
2. 강한 카드를 무제한 사용하는 단순 전략을 막을 수 있다.
3. 목표 가격 밴드 달성과 예산 보전 사이의 긴장감이 생긴다.
4. 완전 성공과 부분 성공을 구분할 수 있어 정산이 풍부해진다.
5. 실제 조작 행위 설명보다 자원 관리형 풍자 게임으로 보이게 된다.
6. “순환 유동성”은 소재의 맛을 살리면서도 직접적 절차 설명을 피할 수 있다.
7. P0 밸런싱 기준이 명확해진다.

### 3.2 Negative Consequences

1. 예산 시스템이 너무 강하면 플레이어가 실험을 꺼릴 수 있다.
2. 예산, 감시도, 변동성, 관심도가 동시에 존재해 초반 학습 부담이 생길 수 있다.
3. 부분 성공 판정이 플레이어에게 실패처럼 느껴질 수 있다.
4. 추상 용어를 사용하면 일부 플레이어에게 카드 효과가 직관적이지 않을 수 있다.
5. 기존 v0.1.1의 `유동성` 표현을 모두 정리해야 한다.

### 3.3 Mitigations

1. 시작 예산은 100으로 충분히 크게 둔다.
2. 최소 유지 예산은 10으로 낮게 둔다.
3. 예산 25% 보전 기준은 완전 성공 기준으로만 두고, 기준 미달은 즉시 실패가 아니라 부분 성공으로 처리한다.
4. 예산을 회복하거나 손실을 줄이는 카드와 이벤트 선택지를 최소 1개 이상 제공한다.
5. 첫 플레이에서는 카드 설명을 수치 중심으로 짧게 보여준다.
6. 정산 화면에서 부분 성공도 명확히 보상하고, 완전 성공으로 개선할 여지를 보여준다.
7. SRS/SDD 단계에서 모든 UI 문구를 안전 표현 원칙에 따라 다시 검토한다.

---

## 4. Alternatives Considered

### 4.1 유동성과 예산을 별도 자원으로 유지

장점은 시장 상태와 플레이어 자원을 분리할 수 있다는 점이다.

하지만 P0에는 자원이 너무 많아지고, 플레이어가 무엇을 관리해야 하는지 흐려질 수 있다.

따라서 P0에서는 유동성을 예산으로 통합한다.

### 4.2 목표 가격 밴드만 성공 조건으로 유지

구현은 단순하다.

하지만 가격만 올리는 전략이 최적해가 되며, 소재가 실제 시세조작 성공 게임처럼 보일 위험이 커진다.

따라서 예산 보전 조건을 함께 둔다.

### 4.3 예산 0 도달 시 실패

일반적인 게임 자원으로는 직관적이다.

하지만 예산이 0이 되기 전까지 무리한 플레이를 해도 된다는 느낌을 줄 수 있다.

따라서 운영자금 고갈의 의미를 살리기 위해 최소 유지 예산 미만을 실패 기준으로 둔다.

### 4.4 “자전거래”를 카드명으로 직접 사용

소재 이해는 빠를 수 있다.

하지만 실제 불법 행위를 직접 연상시키고, 게임이 방법을 가르치는 것처럼 보일 수 있다.

따라서 “순환 유동성”이라는 추상 표현을 사용한다.

### 4.5 예산 보전 기준을 50% 이상으로 설정

완전 성공의 난이도는 높아진다.

하지만 P0 초반에는 지나치게 징벌적일 수 있고, 카드 실험을 막을 수 있다.

따라서 P0 기본 기준은 25%로 둔다.

## 5. Implementation Notes for SRS/SDD

SRS 또는 SDD에서는 이 ADR을 기준으로 다음 항목을 구체화한다.

1. `MarketDefinition.startingBudget`, `minimumBudget`, `successBudgetRatio`의 타입과 기본값
2. 예산 UI 게이지의 색상, 경고 단계, 최소 유지선 표시 방식
3. `순환 유동성` 카드의 쿨다운, 예산 소모량, 가격 압력, 감시도 증가, 변동성 증가 수치
4. 예산 회복 또는 손실 완화 카드의 최소 1개 이상 포함 여부
5. 문서 이벤트 선택지가 예산에 주는 즉시 효과와 지연 효과
6. 완전 성공/부분 성공/실패 판정 순서
7. 점수 계산식에서 예산 보전율의 가중치
8. 정산 화면의 예산 보전율 문구
9. 튜토리얼에서 예산 시스템을 설명하는 안전한 카피
10. 금지 용어와 허용 용어의 UI 문자열 검사 기준

---

## 6. Status

Accepted.

이 ADR은 PRD v0.1.2의 P0 예산 시스템 기준 결정으로 적용한다.
후속 문서에서 이 결정을 변경하려면 새 ADR을 작성하거나 본 ADR을 Superseded 상태로 전환한다.
