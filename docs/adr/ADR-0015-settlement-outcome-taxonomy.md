# ADR-0015 — Settlement Outcome Taxonomy

| 항목 | 내용 |
| --- | --- |
| 문서 종류 | ADR (Architecture Decision Record) |
| 제품명 | Market Manipulator Survival |
| 상태 | Accepted |
| 날짜 | 2026-06-13 |
| 관련 문서 | PRD v0.1.4, ADR-0007, ADR-0008, ADR-0014 |
| 결정 범위 | MVP Day 결과 분류, Final 결과 등급, 사회적 비용 보정 원칙 |

---

## Context

MVP 정산의 핵심 축은 이미 다음으로 확정되었다.

```text
실제 수익 + 감시등급
```

남은 문제는 플레이어가 하루를 마쳤을 때와 5일 Run을 끝냈을 때 결과를 어떤 이름으로 이해하게 할 것인가다.

결과명이 없거나 너무 추상적이면 플레이어는 무엇을 잘했고 무엇을 망쳤는지 알기 어렵다.  
반대로 결과가 너무 많거나 수식 중심이면 PRD 단계에서 과설계가 된다.

---

## Decision

### Day Results

MVP Day 결과는 8개로 제한한다.

1. 완전 성공
2. 위험 성공
3. 고위험 성공
4. 안정 운용
5. 위험 운용
6. 조용한 실패
7. 손실 마감
8. 강제 실패

Day Settlement는 실제 수익과 감시등급을 중심으로 결과를 분류한다.

가격 목표를 달성했더라도 감시등급이 높으면 완전 성공으로 보지 않는다.  
반대로 감시등급이 낮더라도 실제 수익이 부족하면 조용한 실패 또는 안정 운용으로 처리한다.

MVP Day 결과 판정 매트릭스는 다음 방향을 따른다.

| 실제 수익 | 감시등급 A~B | 감시등급 C | 감시등급 D~E | 감시등급 F |
| --- | --- | --- | --- | --- |
| 높음 | 완전 성공 | 위험 성공 | 고위험 성공 | 강제 실패 |
| 보통 | 안정 운용 | 위험 운용 | 위험 운용 | 강제 실패 |
| 낮음 | 조용한 실패 | 조용한 실패 | 위험 운용 | 강제 실패 |
| 손실 | 손실 마감 | 손실 마감 | 손실 마감 | 강제 실패 |

수익 구간의 정확한 수치 기준은 SRS/밸런싱 단계에서 정의한다.

### Final Grades

MVP Final 등급은 6개로 제한한다.

1. S — 조용한 대성공
2. A — 성공적 운용
3. B — 위험한 성공
4. C — 간신히 생존
5. D — 실패한 운용
6. F — 강제 종료

Final Settlement는 5일 Run의 누적 실제 수익, 최종 감시등급, 평균 감시등급, 목표 달성 Day 수, 최종 예산, 최종 보유 비중, 사회적 비용을 기준으로 최종 등급을 산정한다.

| Final 등급 | 조건 감각 |
| --- | --- |
| S — 조용한 대성공 | 누적 수익 높음, 평균 감시 낮음, 강제 실패 없음, 보유 비중 안정 |
| A — 성공적 운용 | 누적 수익 좋음, 감시 관리 가능 |
| B — 위험한 성공 | 누적 수익은 충분하지만 감시 부담이 큼 |
| C — 간신히 생존 | 5일 생존은 했지만 수익/감시/예산 중 하나가 부족 |
| D — 실패한 운용 | 생존했더라도 수익이 낮거나 감시 위험이 높음 |
| F — 강제 종료 | Run 중 강제 실패 발생 |

정확한 점수 계산식과 등급 컷은 SRS/밸런싱 단계에서 정의한다.

### Social Cost

사회적 비용은 개인 참여도 과열, 패닉 발생, 변동성 폭주, 감시 이벤트 누적을 나타내는 보조 정산 지표다.

사회적 비용은 Final 등급을 한 단계 낮추는 보정 요소로 사용할 수 있다.

예시는 다음과 같다.

```text
누적 수익 A급, 감시등급 B지만 사회적 비용이 매우 높음
→ 최종 A가 아니라 B
```

정확한 사회적 비용 계산식과 등급 보정 기준은 SRS/밸런싱 단계에서 정의한다.

---

## Consequences

### Positive

1. 플레이어가 하루 결과와 5일 Run 결과를 직관적으로 이해할 수 있다.
2. 실제 수익과 감시등급의 긴장 관계가 결과명에 드러난다.
3. 고수익 고감시는 성공이지만 위험한 성공으로 분류된다.
4. 저수익 저감시는 완전 성공이 아니라 조용한 실패로 분류된다.
5. 사회적 비용이 최종 결과에 의미 있게 연결된다.

### Negative

1. `위험 운용`과 `고위험 성공`의 차이를 UI에서 명확히 설명해야 한다.
2. 사회적 비용 보정이 불투명하면 등급 하락이 불공정하게 느껴질 수 있다.
3. 정확한 수익 구간을 SRS/밸런싱에서 잘못 잡으면 결과명이 체감과 어긋날 수 있다.

### Mitigations

1. 정산 화면은 결과명과 함께 주요 원인 2~3개를 표시한다.
2. 사회적 비용이 등급을 낮춘 경우 그 사실을 정산 화면에 표시한다.
3. PRD에서는 방향만 고정하고 정확한 컷과 공식은 SRS/밸런싱으로 넘긴다.

---

## Alternatives Considered

### Day 결과 7개

단순하지만 수익은 보통인데 감시가 높은 경우를 표현하기 어렵다.  
`위험 운용`을 추가해 8개로 고정한다.

### Final 결과를 단일 점수로만 표시

계산은 단순하지만 플레이어가 결과의 의미를 파악하기 어렵다.  
S/A/B/C/D/F 등급을 사용한다.

### 사회적 비용을 표시만 하고 등급에 반영하지 않음

설명은 쉬워지지만 개인 참여도 과열, 패닉, 변동성 폭주가 최종 결과에 약하게 연결된다.  
MVP에서는 보조 등급 보정 요소로 둔다.

---

## PRD Impact

PRD v0.1.4는 다음을 반영한다.

1. Day 결과 8개를 명시한다.
2. Day 결과 판정 매트릭스를 추가한다.
3. Final 등급 6개를 명시한다.
4. 사회적 비용을 Final 등급 보정 지표로 정의한다.
5. 정확한 수익 구간, 점수 계산식, 등급 컷은 SRS/밸런싱 단계로 넘긴다.
