# ADR-0026 — Sector and Asset Selection Scope

| 항목 | 내용 |
| --- | --- |
| 상태 | Accepted |
| 날짜 | 2026-06-13 |
| 관련 문서 | PRD v0.1.5, ADR-0003, ADR-0022, ADR-0023, ADR-0024 |

## Context

MVP에는 8개 허구 섹터와 섹터당 3개 허구 종목, 총 24개 종목이 존재한다.

ADR-0023은 종목명과 섹터는 고정하되, 세부 Asset 성향은 Run 시작 시 무작위로 배정하기로 결정했다. 따라서 종목 선택은 고정 난이도 선택이 아니라, 같은 종목을 고르더라도 Run마다 다른 시장 읽기 경험을 만드는 진입점이 된다.

## Decision

MVP에서는 플레이어가 Run 시작 시 섹터와 종목을 직접 선택한다.

모든 섹터와 종목은 MVP에서 처음부터 선택 가능하다.  
별도의 해금 구조는 제공하지 않는다.

다만 선택한 종목의 세부 성향은 Run 시작 시 랜덤으로 배정되며, 플레이어에게 직접 표시하지 않는다.  
플레이어는 Day 진행, 장중 반응, Day 정산 메모, 시장 여파를 통해 해당 Run에서의 종목 성향을 점진적으로 유추한다.

이 구조는 종목별 고정 공략을 어렵게 만들고, 같은 종목을 선택해도 Run마다 다른 시장 읽기 경험을 제공하기 위한 것이다.

## Consequences

MVP의 Run 시작 화면은 섹터와 종목을 직접 선택하는 화면이 된다.

초기 해금, 진행도 게이트, 난이도 잠금 UI는 필요하지 않다.

플레이어는 원하는 테마의 섹터/종목으로 바로 시작할 수 있지만, 실제 성향은 Run 중에 읽어야 한다.

정확한 선택 화면 구성, 브리핑 문구, 기본 선택값은 SRS/UX 단계에서 정의한다.

## Alternatives Considered

### Random Assignment Only

기각한다.  
플레이어가 원하는 섹터 테마를 고르는 재미가 줄어든다.

### Unlock-Based Asset Selection

기각한다.  
MVP에서는 진행도 설계와 보상 구조를 추가하지 않고, 핵심 5일 Run 루프 검증에 집중한다.
