# ADR-0012 — Pre-open Card Scope

| 항목 | 내용 |
| --- | --- |
| 문서 종류 | ADR (Architecture Decision Record) |
| 제품명 | Market Manipulator Survival |
| 상태 | Accepted |
| 날짜 | 2026-06-13 |
| 관련 문서 | PRD v0.1.4, ADR-0005, ADR-0007, ADR-0009, ADR-0011 |
| 결정 범위 | MVP 개장 전 카드 목록, Day당 선택 제한, 개장 승인 흐름 |

---

## Context

MVP의 Day 구조는 아침 뉴스, 시장 브리핑, 개장 전 카드 선택, 개장 승인, 장중 운용, Day Settlement로 구성된다.

아침 뉴스가 단순 정보 표시로 끝나지 않으려면, 플레이어가 개장 전에 뉴스에 대응할 수 있는 최소 선택지가 필요하다.  
다만 MVP에서는 카드 수가 많아지면 밸런싱과 UI 복잡도가 커지므로, 개장 전 선택지는 작게 유지해야 한다.

---

## Decision

MVP 개장 전 카드는 4개로 제한한다.

1. 시장 관찰
2. 사전 포지션 구축
3. 방어 자금 배정
4. 관망

각 Day에서 플레이어는 개장 전 카드 중 최대 1장을 선택할 수 있다.  
카드를 사용하지 않는 선택은 `관망`으로 표현한다.

카드 선택 후 플레이어는 `개장 승인` 버튼 또는 도장 액션을 통해 장중 단계로 진입한다.

---

## Card Roles

| 카드 | 역할 |
| --- | --- |
| 시장 관찰 | 뉴스 효과와 목표 조건을 더 명확히 보여주는 정보 카드 |
| 사전 포지션 구축 | 시작 예산을 소모해 개장 직후 가격 추진력을 얻는 공격적 카드 |
| 방어 자금 배정 | 시작 예산 일부를 방어용으로 묶어 급락/패닉 리스크를 낮추는 방어 카드 |
| 관망 | 아무 카드도 쓰지 않고 예산을 보전한 채 개장하는 선택 |

---

## Consequences

이 결정은 아침 뉴스와 장중 운용 사이에 명확한 의사결정 지점을 만든다.

플레이어는 매 Day 시작 시 다음 중 하나를 선택한다.

1. 정보를 더 볼 것인가
2. 시작부터 밀 것인가
3. 위험을 줄일 것인가
4. 예산을 보전할 것인가

MVP에서는 개장 전 카드의 수치 효과를 정확히 확정하지 않는다.  
구체적인 비용, 보정값, 지속 시간은 SRS 또는 밸런싱 단계에서 정의한다.

---

## Alternatives Considered

### More than 4 pre-open cards

기각한다.  
MVP에서 개장 전 카드가 많으면 아침 뉴스와 카드 조합의 밸런싱 범위가 커진다.

### No pre-open cards

기각한다.  
아침 뉴스가 단순 연출로 끝나고, Papers, Please식 사전 판단 구조가 약해진다.

### Mandatory card use

기각한다.  
플레이어가 예산 보전을 선택할 수 있어야 하므로 `관망`을 명시적 선택지로 둔다.

---

## Follow-up

PRD v0.1.4 또는 후속 버전에 MVP 개장 전 카드 4개를 반영한다.

SRS 단계에서는 각 카드의 비용, 효과, 표시 정보, UI 상태, 개장 승인 흐름을 구체화한다.

---

## PRD Impact

ADR-0005와 ADR-0007의 기존 예시 카드 목록 중 `관심 신호 배치`는 MVP 개장 전 카드에서 제외한다.  
MVP 개장 전 카드 목록은 이 ADR이 우선한다.
