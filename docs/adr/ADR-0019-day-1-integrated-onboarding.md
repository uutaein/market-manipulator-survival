# ADR-0019 — Day 1 Integrated Onboarding

| 항목 | 내용 |
| --- | --- |
| 문서 종류 | ADR (Architecture Decision Record) |
| 제품명 | Market Manipulator Survival |
| 상태 | Accepted |
| 날짜 | 2026-06-13 |
| 관련 문서 | PRD v0.1.4, ADR-0007, ADR-0012, ADR-0014, ADR-0015, ADR-0018 |
| 결정 범위 | MVP 튜토리얼 방식, Day 1 온보딩, 정산 힌트, 재시도 흐름 |

---

## Context

MVP는 5일 Run 구조를 사용하고, 플레이어는 아침 뉴스, 개장 전 카드, 장중 수동 액션, 자동 카드, 문서 이벤트, Day 정산을 순서대로 경험한다.

별도 튜토리얼 모드를 만들면 화면과 상태가 늘어나고, MVP의 핵심 흐름인 “새 Run 시작 → 5일 진행 → 정산 → 다시 시작”이 흐려질 수 있다.

이 게임은 플레이어가 첫 판에서 모든 시스템을 완벽히 이해하기보다, 실수와 정산 피드백을 통해 다음 플레이를 개선하는 구조가 더 잘 맞는다.

---

## Decision

MVP에서는 별도의 튜토리얼 모드를 제공하지 않는다.

대신 Day 1을 인게임 온보딩으로 설계한다.

Day 1은 다음 핵심 흐름을 자연스럽게 경험하게 한다.

1. 아침 뉴스를 읽는다.
2. 개장 전 카드를 고른다.
3. 개장 승인으로 장중에 진입한다.
4. 장중에 수동 액션을 써본다.
5. Day 정산에서 수익과 감시등급을 확인한다.

Day 1에서는 지나치게 복잡한 이벤트 조합을 피하고, 약한 시장 이벤트와 제한된 문서 이벤트를 통해 플레이어가 기본 구조를 이해하도록 한다.

플레이어가 Day 1에서 실수하거나 충분한 수익을 내지 못할 수는 있다.  
다만 MVP에서는 Day 1의 감시도 증가와 가격 붕괴 위험을 완만하게 조정해, 시스템을 이해하기 전에 즉시 실패하는 경험을 줄인다.

---

## Day 1 Guidance

Day 1은 모든 시스템을 설명하지 않는다.

보유 비중, 사회적 비용, Market Board의 세부 의미는 Day 1에서 완전히 이해하지 못해도 된다.  
정산 화면에서 자연스럽게 “다음에는 이것도 봐야 한다”는 힌트를 제공한다.

정산 화면은 단순 결과 표시가 아니라 다음 플레이에 도움이 되는 짧은 힌트를 제공한다.

예시는 다음과 같다.

```text
결과: 위험 성공

수익은 충분했지만 감시등급이 D까지 상승했습니다.
다음 Day에는 과열 해소나 감시 완충 카드를 고려하세요.
```

```text
결과: 조용한 실패

감시등급은 안정적이었지만 실제 수익이 부족했습니다.
뉴스가 호재일 때는 사전 포지션 구축이나 가격 추진을 활용해보세요.
```

---

## Replay Flow

MVP는 재시도 흐름을 짧게 유지한다.

| 상황 | 버튼 |
| --- | --- |
| Day 정산 후 | 다음 Day 진행 |
| Run 실패 후 | 같은 조건으로 다시 시작 / 새 Run 시작 |
| Final 정산 후 | 새 Run 시작 |
| 첫 Run 종료 후 | 다른 개장 전 카드를 시도해보라는 짧은 힌트 |

`같은 조건으로 다시 시작`은 플레이어가 방금 실패한 조건을 바로 실험할 수 있게 하는 재플레이 버튼이다.

---

## Consequences

### Positive

1. 별도 튜토리얼 화면과 튜토리얼 캠페인을 MVP에서 제외할 수 있다.
2. 플레이어가 실제 Run 흐름 안에서 배운다.
3. 정산 피드백이 학습 루프가 된다.
4. 첫 실패가 재시도 욕구로 이어질 수 있다.

### Negative

1. 첫 판에서 모든 시스템을 이해하지 못할 수 있다.
2. 정산 힌트 품질이 낮으면 온보딩이 약해질 수 있다.
3. Day 1 보정이 과하면 튜토리얼처럼 느껴지고, 약하면 억울한 즉시 실패가 발생할 수 있다.

### Mitigations

1. Day 1은 약한 시장 이벤트를 우선 사용한다.
2. Day 1 문서 이벤트는 1개만 거의 확정적으로 발생시킨다.
3. 감시도 증가와 가격 붕괴 위험은 완만하게 조정한다.
4. 실패 가능성 자체는 제거하지 않는다.
5. 정산 화면에 다음 행동 힌트를 짧게 표시한다.

---

## PRD Impact

PRD v0.1.4는 다음을 반영한다.

1. 별도 튜토리얼 모드는 MVP에서 제외한다.
2. Day 1을 인게임 온보딩으로 정의한다.
3. Day 1은 핵심 흐름 4~5개만 체험하게 한다.
4. Day 1은 과도한 즉시 실패를 줄이되 실패 가능성은 유지한다.
5. Day/Final 정산 화면은 짧은 학습 힌트를 제공한다.
