# ADR-0014 — Document Event Scope

| 항목 | 내용 |
| --- | --- |
| 문서 종류 | ADR (Architecture Decision Record) |
| 제품명 | Market Manipulator Survival |
| 상태 | Accepted |
| 날짜 | 2026-06-13 |
| 관련 문서 | PRD v0.1.4, ADR-0006, ADR-0007, ADR-0008, ADR-0011, ADR-0012, ADR-0013 |
| 결정 범위 | MVP 문서 이벤트 수량, 발생 방식, 선택지 구조, 장중 일시정지 규칙 |

---

## Context

Market Manipulator Survival의 MVP 상호작용은 세 축으로 정리된다.

```text
아침 뉴스 / 개장 전 카드 = Papers, Please식 사전 판단
장중 자동 카드 / 수동 액션 = Vampire Survivors식 압박 대응
문서 이벤트 = 두 감성을 연결하는 돌발 의사결정
```

문서 이벤트는 장중에 갑자기 표시되는 거래소 질의서, 감시 보고서, 내부 메모, 커뮤니티 경보 같은 문서형 UI다.

이벤트는 장중 압박을 끊고 플레이어에게 제한된 선택지를 제공해야 한다.  
다만 MVP에서 이벤트 종류와 선택지가 너무 많으면 콘텐츠 제작, 밸런싱, UI 복잡도가 커진다.

---

## Decision

MVP 문서 이벤트는 8개로 제한한다.

1. 이상 흐름 질의서
2. 시장 과열 경보
3. 유동성 경색 보고
4. 커뮤니티 폭주 알림
5. 경쟁 데스크 개입 보고
6. 급락 위험 통지
7. 내부 리스크 메모
8. 마감 전 정리 요청

문서 이벤트는 장중에 조건 기반으로 발생한다.

MVP에서는 한 번에 하나의 이벤트만 표시한다.

MVP에서는 문서 이벤트 발생 시 장중 시간이 일시정지된다.  
플레이어가 선택지를 고르면 효과가 적용되고 장중 시간이 재개된다.

각 문서 이벤트는 3개의 선택지를 가진다.

선택지는 대체로 다음 성격으로 구성한다.

1. 안정 선택지
2. 공격 선택지
3. 회피/관망 선택지

---

## Event Roles

| 이벤트 | 등장 조건 방향 | 역할 |
| --- | --- | --- |
| 이상 흐름 질의서 | 감시도가 높을 때 | 감시 리스크를 직접 다루는 이벤트 |
| 시장 과열 경보 | 개인 참여도 또는 변동성이 높을 때 | 너무 뜨거운 시장을 식힐지, 계속 밀지 선택 |
| 유동성 경색 보고 | 예산 또는 시장 유동성이 낮을 때 | 자원이 부족할 때 발생하는 압박 이벤트 |
| 커뮤니티 폭주 알림 | 개인 참여도 과열 또는 Retail Swarm 밀집 시 | 군중 심리를 어떻게 다룰지 선택 |
| 경쟁 데스크 개입 보고 | 경쟁 압박이 높을 때 | 다른 시장 압력과 경쟁 구도를 표현 |
| 급락 위험 통지 | 가격이 붕괴선에 가까울 때 | 실패 직전의 긴급 선택 |
| 내부 리스크 메모 | 보유 비중이 너무 높을 때 | 보유 비중 리스크를 정산 전에 인지시키는 이벤트 |
| 마감 전 정리 요청 | Day 후반부 또는 마감 직전 | 하루 정산 전에 마지막 전략 선택 유도 |

---

## Choice Structure

각 이벤트의 선택지는 완전히 별개의 미니 시스템이 아니라, 다음 공통 성격을 따른다.

| 선택지 성격 | 방향 |
| --- | --- |
| 안정 선택지 | 감시도/변동성 감소, 예산 또는 가격 추진력 손실 |
| 공격 선택지 | 가격/수익 기회 증가, 감시도/변동성 증가 |
| 회피/관망 선택지 | 즉시 비용 없음, 후속 리스크 증가 |

정확한 수치, 등장 조건, 선택지 문구, 발생 쿨다운, 중복 방지 규칙은 SRS/SDD에서 확정한다.

---

## Consequences

### Positive

1. Papers, Please식 문서 판단과 장중 시장 압박이 연결된다.
2. 장중 플레이가 자동 카드와 수동 버튼만 반복되는 것을 막는다.
3. 핵심 위험 축인 감시도, 과열, 유동성, 군중 심리, 경쟁 압박, 붕괴 위험, 보유 비중, 마감 리스크를 모두 다룬다.
4. 이벤트 중 일시정지로 플레이어가 문서를 읽을 시간을 확보한다.
5. 3선택지 구조로 콘텐츠 제작과 밸런싱 범위를 제한한다.

### Negative

1. 일시정지는 실시간 압박감을 잠시 끊을 수 있다.
2. 이벤트가 너무 자주 뜨면 장중 리듬을 방해할 수 있다.
3. 이벤트 선택지 효과가 불명확하면 플레이어가 문서를 귀찮은 팝업으로 느낄 수 있다.

### Mitigations

1. MVP에서는 한 번에 하나의 이벤트만 표시한다.
2. 정확한 발생 빈도와 쿨다운은 SRS/밸런싱 단계에서 제한한다.
3. 선택지는 안정, 공격, 회피/관망 성격을 유지해 읽기 부담을 줄인다.
4. 이벤트 UI는 문서, 도장, 경고문, 내부 메모 톤을 유지한다.

---

## Alternatives Considered

### Events do not pause intraday time

실시간 압박은 유지되지만 문서를 읽기 어렵고 밸런싱도 복잡해진다.  
MVP에서는 문서 판단 감성을 살리고 구현 부담을 낮추기 위해 일시정지를 채택한다.

### More than 8 document events

콘텐츠 다양성은 늘지만 MVP 제작량과 밸런싱 범위가 커진다.  
핵심 위험 축을 커버하는 8개로 제한한다.

### No document events

구현은 단순하지만 Papers, Please식 문서 판단과 장중 압박이 분리된다.  
게임 정체성을 위해 MVP에 포함한다.

---

## PRD Impact

PRD v0.1.4는 다음을 반영한다.

1. MVP 문서 이벤트 8개를 명시한다.
2. 문서 이벤트는 장중 조건 기반으로 발생한다고 정의한다.
3. 이벤트 발생 시 장중 시간은 일시정지된다.
4. 각 이벤트는 3개 선택지를 가진다.
5. 선택지는 안정, 공격, 회피/관망 성격으로 구성한다.
6. 정확한 수치와 발생 규칙은 SRS/SDD로 넘긴다.
