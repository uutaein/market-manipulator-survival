---

## 1. Context

Market Manipulator Survival의 MVP는 5일짜리 Run으로 구성된다.

각 Day는 다음 흐름을 가진다.

```text
아침 뉴스
→ 시장 브리핑
→ 개장 전 카드
→ 개장 승인
→ 장중 운용
→ Day Settlement
```

5일차 Day Settlement가 끝나면 Final Settlement를 통해 Run 전체 결과를 평가한다.

기존 PRD와 ADR에서는 다음 요소가 이미 확정되었다.

1. 플레이어는 예산을 핵심 생존 자원으로 사용한다.
2. 플레이어는 가격, 보유 비중, 개인 참여도, 시장 유동성, 감시도, 변동성을 관리한다.
3. 각 Day의 목표는 단순 가격 상승이 아니라 목표 가격 밴드에 안착하고 예산을 보전하는 것이다.
4. 감시도 100 도달, 예산 고갈, 치명적 가격 붕괴는 즉시 Run 실패 조건이다.
5. 장중에는 Retail Swarm과 시장 압력 시스템이 가격, 관심, 변동성, 감시 위험을 시각적·수치적으로 표현한다.

남은 핵심 문제는 다음이다.

> 플레이어가 하루 또는 5일 Run을 “잘했다”고 판단하는 기준은 무엇인가?

단순히 가격 목표를 달성했는지만 보면 플레이어는 감시 리스크를 무시하고 강한 액션만 반복할 가능성이 높다.  
반대로 감시도만 낮게 유지하는 것을 성공으로 보면 시장 운용 게임의 성과감이 약해진다.

따라서 MVP 정산은 **실제 수익**과 **감시등급**을 함께 평가해야 한다.

---

## 2. Decision

MVP 정산은 **실제 수익**과 **감시등급**을 이중 축으로 평가한다.

플레이어는 가격 목표를 달성하더라도 감시등급이 높으면 완전 성공으로 평가받지 못한다.  
반대로 감시등급이 낮더라도 실제 수익이 부족하면 조용한 실패 또는 부분 성공으로 평가된다.

Day Settlement와 Final Settlement는 모두 다음 두 축을 중심으로 결과를 산정한다.

| 축 | 의미 |
| --- | --- |
| 실제 수익 | 플레이어가 해당 Day 또는 Run에서 실질적으로 확보한 성과 |
| 감시등급 | 플레이어의 운용이 얼마나 위험하거나 의심스러웠는지 나타내는 리스크 등급 |

보조 지표로는 예산 보전, 보유 비중, 개인 참여도, 변동성, 사회적 비용을 사용한다.

---

## 3. Surveillance Rating

감시도는 장중에 0~100 숫자로 누적된다.

정산 화면에서는 감시도 숫자를 감시등급으로 변환해 보여준다.

| 감시도 | 감시등급 | 의미 | 결과 영향 |
| ---: | --- | --- | --- |
| 0~24 | A — 정상 | 거의 의심 없음 | 완전 성공 가능 |
| 25~49 | B — 관찰 | 약간의 이상 흐름 | 안정 성공 가능 |
| 50~74 | C — 주의 | 감시 대상 가능성 | 위험 성공 이하 |
| 75~94 | D — 위험 | 강한 경고 상태 | 고위험 성공 또는 위험 운용 |
| 95~99 | E — 임박 | 다음 실수 시 강제 실패급 | 매우 큰 패널티 |
| 100 | F — 적발 | 즉시 실패 | Run 실패 |

감시등급은 Day Settlement와 Final Settlement에 모두 반영된다.

감시도 100에 도달하면 Day Settlement를 기다리지 않고 즉시 Run 실패로 처리한다.

---

## 4. Profit Model

MVP에서 실제 수익은 복잡한 회계 시스템이 아니라 게임용 추상 점수로 계산한다.

실제 수익은 다음 요소를 반영한다.

1. 대상 종목의 가격 성과
2. 플레이어의 보유 비중
3. 포지션 정리 여부
4. 잔여 예산
5. 미정리 보유 비중 패널티

P0 수준의 개념 공식은 다음과 같다.

```text
실제 수익 점수 =
  가격 성과 × 보유 비중 보정
  + 포지션 정리 보너스
  + 잔여 예산 보너스
  - 미정리 보유 비중 패널티
  - 과열/붕괴 패널티
```

정확한 수치와 가중치는 SRS 또는 밸런싱 단계에서 확정한다.

중요한 원칙은 다음이다.

1. 단순 가격 상승률만으로 수익을 계산하지 않는다.
2. 보유 비중이 너무 낮으면 가격을 올려도 실질 수익이 제한된다.
3. 보유 비중이 너무 높으면 장중 영향력은 커지지만 마감 정산 리스크가 증가한다.
4. 포지션을 적절히 정리하면 실제 수익과 예산 회복에 긍정적 영향을 준다.
5. 미정리 보유 비중이 과도하면 장부상 성과만 있고 실제 수익은 낮게 평가된다.

## 5. Day Settlement

Day Settlement는 하루 장 마감 결과를 평가한다.

Day Settlement의 주요 입력값은 다음과 같다.

| 항목 | 설명 |
| --- | --- |
| 종가 위치 | 목표 가격 밴드 안에 들어왔는지 |
| 실제 수익 | 해당 Day에서 확보한 수익 점수 |
| 감시등급 | 해당 Day 마감 시 감시도 기반 등급 |
| 예산 보전 | 시작 예산 대비 잔여 예산 |
| 보유 비중 | 마감 시 플레이어의 보유 비중 |
| 개인 참여도 | 과열 또는 무관심 상태 여부 |
| 변동성 | 장 마감 시 시장 안정도 |
| 사회적 비용 | 과열, 패닉, Retail Swarm 폭주 등의 누적 영향 |

Day 결과는 실제 수익과 감시등급의 조합으로 분류한다.

| 실제 수익 | 감시등급 | Day 결과 |
| --- | --- | --- |
| 높음 | A~B | 완전 성공 |
| 높음 | C | 위험 성공 |
| 높음 | D~E | 고위험 성공 |
| 보통 | A~C | 안정 운용 |
| 보통 | D~E | 위험 운용 |
| 낮음 | A~B | 조용한 실패 |
| 낮음 | C~E | 실패한 운용 |
| 손실 | A~E | 손실 마감 |
| 무관 | F | 적발 / Run 실패 |

Day Settlement는 반드시 다음 Day에 영향을 줄 수 있다.

예시는 다음과 같다.

1. 감시등급이 높으면 다음 Day의 시작 감시도 또는 감시 민감도가 증가할 수 있다.
2. 실제 수익이 높으면 다음 Day 시작 예산이 증가할 수 있다.
3. 손실 마감이면 다음 Day 시작 예산이 감소한다.
4. 개인 참여도가 과열된 채로 끝나면 다음 Day에 패닉 또는 규제 뉴스 확률이 증가할 수 있다.
5. 보유 비중이 과도하게 높으면 다음 Day에 포지션 정리 압박이 발생할 수 있다.

정확한 이월 규칙은 SRS에서 확정한다.

---

## 6. Final Settlement

Final Settlement는 5일 Run 전체 결과를 평가한다.

Final Settlement의 주요 입력값은 다음과 같다.

| 항목 | 설명 |
| --- | --- |
| 누적 실제 수익 | 5일 동안 확보한 총 수익 점수 |
| 최종 예산 | 5일차 마감 후 남은 예산 |
| 최종 감시등급 | Run 종료 시 감시도 기반 등급 |
| 평균 감시등급 | 5일 동안의 평균 감시 수준 |
| 목표 달성 Day 수 | 목표 밴드 또는 Day 성과 기준을 달성한 일수 |
| 최종 보유 비중 | Run 종료 시 미정리 보유 비중 |
| 사회적 비용 | 과열, 패닉, 개인 참여도 폭주, 변동성 누적 |
| 강제 실패 여부 | 감시도 100, 예산 고갈, 가격 붕괴 등 |

Final 등급은 다음 방향으로 산정한다.

| 최종 등급 | 의미 |
| --- | --- |
| S | 높은 수익과 낮은 감시를 모두 달성한 조용한 대성공 |
| A | 좋은 수익과 관리 가능한 감시를 달성한 성공적 운용 |
| B | 수익은 충분하지만 감시 부담이 있는 위험한 성공 |
| C | 생존은 했지만 수익 또는 감시 관리 중 하나가 부족한 결과 |
| D | 낮은 수익, 높은 감시, 또는 불안정한 운용 |
| F | Run 중 적발, 파산, 치명적 붕괴로 강제 종료 |

Final Settlement는 플레이어에게 다음 정보를 명확히 보여줘야 한다.

```text
누적 실제 수익
최종 감시등급
평균 감시등급
목표 달성 Day 수
최종 예산
최종 보유 비중
사회적 비용
최종 등급
```

## 7. Immediate Failure Conditions

MVP의 즉시 Run 실패 조건은 다음으로 확정한다.

1. 예산이 최소 유지선 미만으로 하락
2. 감시도 100 도달
3. 가격이 치명적 붕괴선 아래로 이탈

이 조건이 발생하면 해당 Day의 일반 정산을 기다리지 않고 Run 실패로 처리한다.

P0에서는 즉시 실패 조건을 과도하게 늘리지 않는다.  
보유 비중 0, 개인 참여도 100, 변동성 100 같은 상태는 직접 실패 조건이 아니라 패널티와 위험 증폭 요소로 사용한다.

---

## 8. Consequences

이 결정의 긍정적 결과는 다음과 같다.

1. 플레이어는 단순히 가격을 올리는 것만으로 완전 성공을 얻을 수 없다.
2. 고수익 고감시는 위험 성공으로 분류되어 게임의 풍자적 긴장감을 강화한다.
3. 저수익 저감시는 조용한 실패로 분류되어 지나치게 보수적인 플레이도 제한된다.
4. 예산, 보유 비중, 개인 참여도, 감시도, 변동성이 모두 정산에 의미 있게 연결된다.
5. Day 결과가 다음 Day에 영향을 줄 수 있어 5일 Run 구조가 강화된다.
6. 실제 금융 범죄 절차를 설명하지 않고도 “성과와 리스크의 균형”이라는 게임성을 만들 수 있다.

부정적 결과와 리스크는 다음과 같다.

1. 정산 기준이 너무 복잡하면 플레이어가 결과를 이해하기 어려울 수 있다.
2. 실제 수익 계산식이 불투명하면 결과가 불공정하게 느껴질 수 있다.
3. 감시등급 패널티가 너무 크면 플레이어가 적극적 액션을 꺼릴 수 있다.
4. 수익 보상이 너무 크면 감시 리스크가 무시될 수 있다.

대응 방향은 다음과 같다.

1. 정산 화면에서 “왜 이 결과가 나왔는지”를 항목별로 보여준다.
2. P0에서는 수익 계산식을 단순하게 유지한다.
3. 감시등급과 수익 점수의 밸런스는 플레이 테스트 후 조정한다.
4. 플레이어에게 Day별 피드백 문구를 제공해 다음 Day 전략을 유도한다.

## 9. Alternatives Considered

### 9.1 가격 목표만으로 성공 판정

가격이 목표 밴드에 들어오면 성공으로 보는 방식이다.

채택하지 않은 이유는 다음과 같다.

1. 감시도와 예산 관리의 의미가 약해진다.
2. 플레이어가 강한 가격 추진 액션만 반복할 가능성이 높다.
3. 게임의 풍자적 핵심인 성과와 리스크의 균형이 사라진다.

### 9.2 감시도만으로 리스크 판정

감시도 100만 피하면 성공으로 보는 방식이다.

채택하지 않은 이유는 다음과 같다.

1. 실제 수익과 성과감이 약해진다.
2. 지나치게 보수적인 플레이가 유리해진다.
3. 시장 운용 게임으로서의 목표가 흐려진다.

### 9.3 복잡한 회계 기반 수익 계산

실현 손익, 평가 손익, 수수료, 보유량, 거래 단가를 모두 상세 계산하는 방식이다.

채택하지 않은 이유는 다음과 같다.

1. MVP 범위에 비해 구현과 밸런싱 비용이 높다.
2. 게임이 풍자형 서바이벌이 아니라 복잡한 투자 시뮬레이터처럼 보일 수 있다.
3. 실제 금융 거래 재현에 가까워져 안전한 추상화 원칙과 맞지 않는다.

### 9.4 단일 점수만 표시

수익, 감시, 예산, 보유 비중을 모두 합산해 하나의 점수만 보여주는 방식이다.

채택하지 않은 이유는 다음과 같다.

1. 플레이어가 무엇을 잘했고 무엇을 실패했는지 알기 어렵다.
2. 감시 리스크와 실제 수익의 긴장 관계가 약해진다.
3. Day별 전략 피드백으로 활용하기 어렵다.

## 10. PRD Update Requirements

다음 PRD 갱신 시 아래 내용을 반영해야 한다.

1. 정산 구조 섹션 추가
2. 실제 수익 정의 추가
3. 감시등급 표 추가
4. Day Settlement 결과 분류표 추가
5. Final Settlement 등급표 추가
6. 즉시 실패 조건 확정
7. Day 결과가 다음 Day에 영향을 줄 수 있다는 방향 추가
8. SRS에서 수익 계산식과 정산 가중치를 확정한다는 미확정 사항 추가

PRD 내 기존 “성공/실패 조건”은 다음과 같이 수정한다.

```text
MVP의 성공 평가는 단순 목표 가격 달성이 아니라 실제 수익과 감시등급을 함께 평가한다.
가격 목표 달성, 예산 보전, 보유 비중 정리, 감시등급, 사회적 비용이 Day 및 Final 정산에 반영된다.
```

## 11. Open Questions for SRS

SRS에서 확정해야 할 항목은 다음과 같다.

1. 실제 수익 점수의 정확한 계산식
2. 수익 높음/보통/낮음/손실의 기준값
3. 감시도 이월 규칙

## 12. Safety and Tone Notes

정산 시스템은 실제 금융 거래, 시세조종, 자전거래, 불법 수익화 절차를 설명하지 않는다.

실제 수익은 게임용 추상 점수이며, 보유 비중과 포지션 정리는 실제 거래 방법이 아니라 게임 내 리스크·성과 지표로 사용한다.

감시등급은 법률·금융 규제의 실제 판정 체계가 아니라, 허구 시장의 풍자적 리스크 시스템이다.

UI와 문서에서는 직접적인 금융 범죄 표현을 피하고 다음 표현을 우선 사용한다.

| 피할 표현 | 사용할 표현 |
| --- | --- |
| 주가조작 | 시장 압력 관리 |
| 자전거래 | 유동성 공급 / 유동성 순환 |
| 개미 | 개인 참여자 / 개인 참여도 |
| 세력 | 운용 데스크 |
| 물량 털기 | 포지션 정리 |
| 펌핑 | 가격 추진 |
| 덤핑 | 과열 해소 / 정산 |
