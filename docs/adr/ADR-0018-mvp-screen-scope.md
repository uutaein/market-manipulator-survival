# ADR-0018 — MVP Screen Scope

| 항목 | 내용 |
| --- | --- |
| 문서 종류 | ADR (Architecture Decision Record) |
| 제품명 | Market Manipulator Survival |
| 상태 | Accepted |
| 날짜 | 2026-06-13 |
| 관련 문서 | PRD v0.1.4, ADR-0007, ADR-0012, ADR-0014, ADR-0015 |
| 결정 범위 | MVP 화면 목록, 화면 흐름, 제외 화면, 문서 이벤트 모달 처리 |

---

## Context

MVP의 기능 단위가 닫히면서, 플레이어가 실제로 어떤 화면을 어떤 순서로 보게 되는지 확정할 필요가 생겼다.

목표는 5일 Run의 시작부터 최종 정산까지 끊기지 않는 최소 화면 흐름을 만드는 것이다.

---

## Decision

MVP 화면은 8개로 제한한다.

1. 메인 메뉴
2. Run 시작 / 종목 선택
3. 아침 뉴스 / 시장 브리핑
4. 개장 전 카드 선택
5. 장중 운용 화면
6. 문서 이벤트 팝업
7. Day 정산 화면
8. Final 정산 화면

문서 이벤트는 별도 전체 화면이 아니라 장중 운용 화면 위에 표시되는 모달 팝업으로 처리한다.

MVP 화면 흐름은 다음과 같다.

```text
메인 메뉴
→ Run 시작 / 종목 선택
→ Day 1 아침 뉴스 / 시장 브리핑
→ 개장 전 카드 선택
→ 개장 승인
→ 장중 운용 화면
→ 문서 이벤트 팝업
→ Day 정산
→ Day 2 아침 뉴스
→ ...
→ Day 5 정산
→ Final 정산
```

---

## Excluded Screens

MVP에서는 다음 화면을 포함하지 않는다.

| 화면 | 처리 |
| --- | --- |
| 상세 도감 | P1 |
| 카드 컬렉션 | P1 |
| 섹터별 통계 | P1 |
| 리플레이 | P2 |
| 설정 고급 화면 | P1 |
| 튜토리얼 전용 캠페인 | P1 |
| 온라인 랭킹 | 제외 또는 P2 |
| 전체 24개 종목 상세 시장판 | 제외 |

---

## Consequences

### Positive

1. MVP 화면 범위가 구현 가능한 수준으로 작아진다.
2. 5일 Run 흐름이 끊기지 않는다.
3. 문서 이벤트가 장중 흐름과 연결된다.
4. 도감, 컬렉션, 통계 같은 부가 화면이 MVP 범위를 침범하지 않는다.

### Negative

1. 카드/섹터/종목 정보를 깊게 탐색하는 화면은 MVP에 없다.
2. 설정은 기본 수준으로 제한된다.
3. 튜토리얼 전용 화면 없이 온보딩을 게임 안에 녹여야 한다.

---

## PRD Impact

PRD v0.1.4는 다음을 반영한다.

1. MVP 화면 8개를 명시한다.
2. 문서 이벤트를 장중 모달로 정의한다.
3. 도감, 카드 컬렉션, 섹터별 통계, 리플레이, 온라인 랭킹, 고급 설정 화면을 MVP에서 제외한다.
