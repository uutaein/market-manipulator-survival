## 1. Context

사용자는 장중에 Vampire Survivors처럼 인터랙티브하게 움직이는 시스템을 원했다.  
구체적으로는 유동량 늘리기, 가격 올리기, 가격 내려오기, 본인 지분, 개인 투자자 지표 등이 필요하다고 제안했다.

또한 개인 투자자가 몰려오는 모습을 주가 변동과 함께 시각적으로 표시하고, 좀비가 몰려오는 것처럼 압박감을 주기를 원했다.

다만 실제 금융 범죄처럼 느껴지지 않도록 직접 표현은 피하고, 추상화된 게임 용어와 수치 시스템으로 표현해야 한다.

---

## 2. Decision

### 2.1 장중 핵심 지표를 7개로 정한다

P0 장중 핵심 지표는 다음과 같다.

| 지표 | 역할 |
| --- | --- |
| 예산 | 카드와 액션 비용, 실패 조건, 정산 기준 |
| 가격 | 목표 밴드 판정 기준 |
| 보유 비중 | 대상 종목에 대한 플레이어 데스크의 영향력 |
| 개인 참여도 | 군중 심리와 Retail Swarm 밀도 |
| 시장 유동성 | 가격 반응성과 거래 흐름 |
| 감시도 | 100 도달 시 실패 |
| 변동성 | 랜덤 흔들림과 급락 위험 |

### 2.2 직접 표현은 추상 게임 용어로 치환한다

다음 용어 치환을 UI와 사용자-facing 문구에 적용한다.

| 직접 표현 | 추상 표현 |
| --- | --- |
| 자전거래 | 유동성 공급 / 순환 유동성 |
| 유동량 늘리기 | 거래 흐름 확대 / 유동성 공급 |
| 주가 끌어올리기 | 가격 추진 |
| 주가 내려오기 | 과열 해소 / 가격 안정화 |
| 본인 지분 | 보유 비중 / 시장 영향력 |
| 개미 지표 | 개인 참여도 |
| 물량 털기 | 포지션 정리 |
| 매집 | 보유 비중 확대 |
| 세력 | 운용 데스크 |

### 2.3 장중 조작은 자동 카드와 제한된 수동 액션을 조합한다

P0에서는 다음 구조를 사용한다.

1. 자동 카드는 일정 주기로 발동한다.
2. 수동 액션은 플레이어가 직접 누르며 쿨다운과 예산 비용을 가진다.
3. 수동 액션은 3~5개로 제한한다.
4. 모든 효과 설명은 수치 중심으로 표시한다.

추천 수동 액션은 다음과 같다.

| 액션 | 역할 |
| --- | --- |
| 가격 추진 | 예산 소모, 단기 상승 압력 |
| 유동성 공급 | 예산 소모, 거래 흐름과 변동성 증가 |
| 과열 해소 | 압력 감소, 감시도/변동성 완화 |
| 방어 자금 투입 | 예산 소모, 급락 방어 |
| 포지션 정리 | 보유 비중 감소, 예산 회복, 가격 압력 감소 |

### 2.4 보유 비중은 영향력과 정산 리스크를 함께 만든다

보유 비중이 높을수록 가격 추진과 방어 효율은 증가한다.  
대신 감시도 증가량과 마감 정산 리스크도 증가한다.

P0 구간은 다음을 기준으로 한다.

| 보유 비중 | 상태 | 효과 방향 |
| --- | --- | --- |
| 0~10% | 영향력 낮음 | 가격 카드 효율 감소 |
| 10~30% | 정상 | 보정 없음 |
| 30~50% | 강한 영향력 | 가격 카드 효율 증가, 감시도 증가 |
| 50% 이상 | 과점 상태 | 효율은 높지만 감시도와 정산 리스크 증가 |

### 2.5 개인 참여도는 Retail Swarm으로 시각화한다

P0 장중에는 개인 참여도를 차트 주변 또는 시장 보드 영역에 나타나는 아이콘/토큰 기반 군집 웨이브로 표시한다.

Retail Swarm은 다음 상태를 가진다.

| 상태 | 설명 |
| --- | --- |
| 조용함 | 참여가 적고 가격 반응이 약함 |
| 관심 형성 | 정상적인 관심과 유입 |
| 과열 | 군집 밀도가 높고 감시도·변동성 증가 |
| 패닉 | 군집이 반대 방향으로 이탈하며 하락 압력 발생 |

### 2.6 Retail Swarm은 실제 게임 상태와 연결한다

Retail Swarm은 단순 배경 연출이 아니다.

1. 개인 참여도가 상승하면 군집 수, 속도, 밀도가 증가한다.
2. 과열 군집은 감시도와 변동성을 증가시킨다.
3. 패닉 군집은 가격 하락 압력을 만든다.
4. 군집 변화는 가격과 뉴스, 카드 효과와 연결된다.

### 2.7 P0 구현은 아이콘/토큰 기반으로 제한한다

P0에서는 실사형 군중, 복잡한 유닛 AI, 고비용 애니메이션을 만들지 않는다.

대신 다음을 구현한다.

1. 개인 참여도에 따른 스폰량 변화
2. 군집 이동 방향 변화
3. 과열 상태의 색상/깜빡임 변화
4. 패닉 상태의 빠른 이탈
5. 차트와 연결되는 간단한 시각 피드백

---

## 3. Consequences

### 3.1 Positive Consequences

1. 장중 화면이 숫자 패널에 머무르지 않고 살아 움직인다.
2. Vampire Survivors식 웨이브 압박을 시장 테마에 맞게 변환할 수 있다.
3. 개인 참여도 수치가 직관적으로 이해된다.
4. 플레이어가 과열과 패닉을 시각적으로 판단할 수 있다.
5. 가격 목표 밴드 안착이 더 인터랙티브해진다.
6. 직접적 금융 표현 없이도 소재의 맛을 유지할 수 있다.

### 3.2 Negative Consequences

1. 장중 화면의 시각 복잡도가 증가한다.
2. Retail Swarm과 차트가 서로 가릴 수 있다.
3. 군집 연출과 실제 수치 변화가 어긋나면 혼란을 줄 수 있다.
4. 장중 액션이 많아지면 P0 밸런싱 비용이 커진다.

---

## 4. Alternatives Considered

### 4.1 개인 참여도를 숫자로만 표시

구현은 쉽지만 뱀서식 압박감이 약하다.  
사용자가 원하는 “좀비처럼 몰려오는” 체감과 맞지 않아 채택하지 않는다.

### 4.2 사람 캐릭터 기반 군중 구현

표현력은 높지만 아트·애니메이션·성능 비용이 크다.  
P0에서는 아이콘/토큰 기반 추상 표현으로 제한한다.

### 4.3 수동 액션 없이 자동 카드만 사용

Vampire Survivors와 유사하지만 시장 조절의 직접성이 약하다.  
P0에서는 자동 카드와 제한된 수동 액션을 함께 사용한다.

---

## 5. Follow-ups

1. Retail Swarm의 정확한 스폰 공식은 SRS/SDD에서 확정한다.
2. 가격 Tick Formula의 기본 계수는 SRS/SDD에서 확정한다.
3. 수동 액션 3~5개의 최종 목록과 수치 효과는 SRS/SDD에서 확정한다.
4. Retail Swarm이 차트를 가리지 않도록 UI 레이아웃을 프로토타입에서 검증한다.
