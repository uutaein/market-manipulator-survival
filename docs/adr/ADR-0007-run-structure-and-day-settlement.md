## 1. Context

Market Manipulator Survival의 MVP 구조는 다음 핵심 요소를 이미 포함한다.

1. 개장 전 아침 뉴스와 시장 브리핑
2. 개장 전 카드 선택
3. 개장 승인 액션
4. 장중 가격 변동과 시장 보드
5. 예산, 보유 비중, 개인 참여도, 시장 유동성, 감시도, 변동성 관리
6. Retail Swarm 기반 개인 참여 군집 시각화
7. 데이 종료 후 성과 정산

초기 문서에서는 “라운드”라는 표현이 사용되었으나, 게임의 핵심 구조가 `아침 뉴스 → 개장 → 마감` 흐름을 가지므로 단일 라운드를 게임 전체 종료 단위로 보면 확장성과 몰입감이 부족하다.

특히 Papers, Please식 감성을 살리려면 매일 다른 문서와 조건을 확인하고, 그날의 장을 열고, 마감 후 다음 날로 넘어가는 반복 구조가 필요하다.

따라서 MVP에서 “라운드”가 무엇을 의미하는지, 한 판의 게임이 몇 개의 Day로 구성되는지, Day 정산과 최종 정산의 차이를 명확히 결정해야 한다.

---

## 2. Decision

MVP의 게임 한 판은 **5일짜리 Run**으로 구성한다.

각 Day는 하나의 시장 세션이며, 다음 흐름을 가진다.

```text
아침 뉴스
→ 시장 브리핑
→ 개장 전 카드 선택
→ 개장 승인
→ 장중 운용
→ 마감
→ Day Settlement
```

5일차 Day Settlement가 끝나면 **Final Settlement**를 진행한다.

용어는 다음과 같이 구분한다.

| 용어 | 의미 |
| --- | --- |
| Day / 장 | 하루 시장 세션. 아침 뉴스부터 마감 정산까지의 단위 |
| Run / 런 | 여러 Day로 구성된 게임 한 판 |
| Day Settlement / 데이 정산 | 하루 장 마감 후 해당 Day의 성과와 리스크를 정리하는 중간 정산 |
| Final Settlement / 최종 정산 | 5일 Run 종료 후 전체 성과를 평가하는 최종 정산 |
| Immediate Run Failure / 즉시 런 실패 | Day 도중 치명 조건이 발생해 Run 전체가 종료되는 상태 |

MVP에서는 다음 기본값을 사용한다.

| 항목 | MVP 결정 |
| --- | --- |
| Run 길이 | 5 Day |
| Day 장중 시간 | 기본 6분 |
| 개장 전 단계 제한 시간 | 없음 |
| 개장 전 카드 사용 | Day당 최대 1장 |
| 조기 성공 종료 | 없음 |
| Day 종료 | 장중 시간 종료 시 마감 |
| 즉시 실패 조건 | 예산 고갈, 감시도 100, 치명적 가격 붕괴 등 |
| 최종 정산 | 5일차 마감 후 진행 |

## 3. Day Structure

### 3.1 아침 뉴스

각 Day는 아침 뉴스 화면으로 시작한다.

아침 뉴스는 시장 전체, 특정 섹터, 특정 종목 중 하나 이상의 범위에 영향을 줄 수 있다.

뉴스는 해당 Day의 초기 조건, 관심도, 변동성, 가격 압력, 감시 민감도, 예산 효율 등에 영향을 준다.

### 3.2 시장 브리핑

시장 브리핑은 플레이어에게 다음 정보를 제공한다.

1. 오늘의 주요 뉴스
2. 영향을 받는 섹터 또는 종목
3. 플레이어 대상 종목의 기본 프로필
4. Today Condition
5. 목표 가격 밴드
6. 예산 보전 기준
7. 주요 리스크

### 3.3 개장 전 카드

플레이어는 개장 전 카드 중 최대 1장을 선택할 수 있다.

개장 전 카드는 Day 시작 조건을 조정한다.

예시는 다음과 같다.

| 카드 | 역할 |
| --- | --- |
| 시장 관찰 | 뉴스 효과와 목표 조건을 더 명확히 보여주는 정보 카드 |
| 사전 포지션 구축 | 예산 일부를 사용해 개장 직후 가격 압력 확보 |
| 방어 자금 배정 | 급락 방어 기반 확보 |
| 관망 | 아무 카드도 사용하지 않고 예산 보전 |

### 3.4 개장 승인

플레이어가 개장 승인 버튼 또는 도장 액션을 수행하면 장중 단계로 전환된다.

개장 승인은 Papers, Please식 문서/도장 감성을 살리는 핵심 UX 연출로 사용한다.

### 3.5 장중 운용

장중 단계에서 플레이어는 다음 지표를 관리한다.

1. 가격
2. 예산
3. 보유 비중
4. 개인 참여도
5. 시장 유동성
6. 감시도
7. 변동성

장중에는 플레이어 대상 종목이 상세 계산되고, 비플레이어 종목은 시장 보드에서 간략 등락률로 표시된다.

### 3.6 Day Settlement

Day가 종료되면 Day Settlement를 진행한다.

Day Settlement는 게임 전체 종료가 아니라 다음 Day로 이어지는 중간 정산이다.

Day Settlement는 다음 항목을 평가한다.

| 항목 | 설명 |
| --- | --- |
| 종가 | 목표 가격 밴드 안착 여부 |
| 예산 | 시작 예산 대비 잔여 예산 |
| 보유 비중 | 과도한 보유 비중 여부 |
| 감시도 | 다음 Day로 이월될 위험 수준 |
| 개인 참여도 | 과열 또는 침체 상태 |
| 시장 유동성 | 다음 Day의 시장 반응 기반 |
| 사회적 비용 | 과열, 패닉, 감시 이벤트로 누적된 부정적 비용 |

---

## 4. Settlement Model

### 4.1 Day Settlement Result

Day Settlement는 단순 성공/실패보다 등급형 결과를 사용한다.

MVP 후보 결과는 다음과 같다.

| 결과 | 조건 |
| --- | --- |
| 안정 성공 | 목표 가격 밴드 안착, 예산 보전, 감시도 안정 |
| 위험 성공 | 목표 가격 밴드는 달성했지만 감시도 또는 변동성이 높음 |
| 비용 과다 | 목표는 달성했지만 예산 보전 기준 미달 |
| 부분 실패 | Day를 생존했지만 목표 가격 밴드 미달 |
| 위기 종료 | 다음 Day에 큰 불이익을 남기는 상태로 마감 |
| 즉시 실패 | Day 도중 치명 조건으로 Run 종료 |

정확한 수치 기준은 SRS에서 정의한다.

### 4.2 Final Settlement

5일차 종료 후 Final Settlement를 진행한다.

Final Settlement는 다음 항목을 기반으로 전체 Run 성과를 평가한다.

1. 누적 예산
2. 누적 성과
3. 목표 달성 Day 수
4. 잔여 보유 비중
5. 누적 감시도
6. 누적 사회적 비용
7. 강제 실패 여부
8. 위험 성공 비율

Final Settlement는 플레이어에게 “얼마나 벌었는가”뿐 아니라 “얼마나 위험하고 비용이 큰 방식으로 운영했는가”를 함께 보여준다.

## 5. Rationale

### 5.1 Papers, Please식 반복 구조와 잘 맞음

게임의 핵심 감성은 매일 문서를 읽고, 조건을 판단하고, 승인/거절 또는 개장 결정을 내리는 압박감이다.

5일 Run 구조는 이 감성을 자연스럽게 반복시킨다.

### 5.2 한 판의 길이를 MVP 검증에 맞게 제한

Day당 장중 시간을 6분으로 두면 5일 Run은 약 30분 내외가 된다.

이는 짧은 프로토타입보다 충분한 전략성을 제공하면서도, 10분 × 5일처럼 지나치게 길어지는 문제를 피한다.

### 5.3 시스템 간 연결성을 강화

Day Settlement가 있으면 다음 시스템들이 자연스럽게 연결된다.

1. 아침 뉴스
2. 종목 프로필
3. Today Condition
4. 개장 전 카드
5. 장중 인터랙션
6. Retail Swarm
7. 예산 보전
8. 감시도 누적
9. 최종 정산

### 5.4 성공/실패를 더 풍부하게 표현

단일 라운드 성공/실패보다 Day별 중간 결과와 최종 결과를 분리하면 플레이어가 매일 다른 전략을 시도할 수 있다.

목표를 달성했지만 예산을 과도하게 소모한 Day, 목표를 놓쳤지만 리스크를 낮춘 Day 등 다양한 결과를 표현할 수 있다.

## 6. Alternatives Considered

### 6.1 단일 10분 라운드 = 게임 한 판

장점은 구현이 가장 단순하다는 점이다.

하지만 아침 뉴스, 개장 전 카드, Day 정산 구조가 반복되지 않아 게임의 정체성이 약해진다.

또한 Papers, Please식 매일 다른 문서와 조건을 처리하는 감성이 줄어든다.

결론: MVP의 주 구조로 채택하지 않는다.

### 6.2 무한 Day 진행

Day를 계속 이어가는 로그라이크 구조도 가능하다.

하지만 MVP에서는 밸런싱과 종료 조건이 복잡해지고, 플레이 테스트 시간이 길어진다.

결론: P1 이후 모드 후보로 둔다.

### 6.3 3일 Run

짧고 빠른 검증에는 유리하다.

하지만 종목 프로필, 뉴스, 개장 전 카드, 장중 운용, 정산의 변화를 충분히 경험하기에는 다소 짧다.

결론: 튜토리얼 또는 쉬운 난이도 후보로 둔다.

### 6.4 10일 Run

캠페인 감각은 강해진다.

하지만 MVP 검증에는 길고, 콘텐츠 수량과 밸런싱 부담이 커진다.

결론: P1 또는 캠페인 모드 후보로 둔다.

## 7. Consequences

### 7.1 Positive Consequences

1. 게임 한 판의 구조가 명확해진다.
2. 아침 뉴스와 개장 전 카드가 반복 가능한 핵심 루프로 자리 잡는다.
3. Day별 성과와 최종 성과를 분리할 수 있다.
4. 감시도, 사회적 비용, 보유 비중 같은 누적 리스크를 설계하기 쉬워진다.
5. MVP 플레이 시간이 대략 30분 내외로 예측 가능해진다.

### 7.2 Negative Consequences

1. 단일 라운드 게임보다 상태 관리가 복잡해진다.
2. Day 간 이월되는 값과 초기화되는 값을 명확히 정의해야 한다.
3. 콘텐츠 반복을 줄이기 위해 뉴스, 종목, 카드, 이벤트 수량이 필요하다.
4. Final Settlement UI가 추가로 필요하다.

### 7.3 Mitigations

1. P0에서는 Run 길이를 5일로 고정한다.
2. P0에서는 Day 장중 시간을 6분으로 고정한다.
3. P0에서는 개장 전 카드 사용을 Day당 1장으로 제한한다.
4. Day 간 이월 값은 SRS에서 명시적으로 정의한다.
5. Final Settlement는 간단한 결과표와 등급 중심으로 구현한다.

## 8. PRD Update Requirements

다음 PRD 버전에는 아래 항목을 반영해야 한다.

1. “라운드” 표현을 “Day”와 “Run”으로 정리한다.
2. MVP는 5일 Run으로 구성한다고 명시한다.
3. Day 기본 흐름을 `아침 뉴스 → 시장 브리핑 → 개장 전 카드 → 개장 승인 → 장중 운용 → Day Settlement`로 정의한다.
4. 5일차 종료 후 Final Settlement를 진행한다고 명시한다.
5. Day 장중 시간의 MVP 기본값을 6분으로 둔다.
6. 조기 성공 종료는 MVP 범위에서 제외한다.
7. 즉시 실패 조건은 Run 전체 실패로 정의한다.
8. Day Settlement와 Final Settlement의 평가 항목을 구분한다.
9. P1 후보로 3일 튜토리얼 Run, 10일 캠페인 Run, 무한 Run을 남긴다.

아래 항목은 ADR에서 방향만 정하고, SRS에서 수치와 상태 전이를 확정한다.

1. Day 간 이월되는 값과 초기화되는 값
2. Day Settlement 결과 등급의 정확한 기준
3. Final Settlement 등급 산정식
4. 감시도 누적/감소 규칙
5. 보유 비중의 Day 간 변화 규칙
6. 사회적 비용 산정 기준
7. 가격 밴드 유지 시간 요구 여부
8. Day 실패와 Run 실패의 경계 조건
9. 튜토리얼 모드 제공 여부

## 10. Final Decision Summary

MVP는 **5일 Run**으로 구성한다.

각 Day는 **아침 뉴스 → 시장 브리핑 → 개장 전 카드 → 개장 승인 → 장중 운용 → Day Settlement** 흐름을 가진다.

Day Settlement는 게임 전체 종료가 아니라 다음 Day로 이어지는 중간 정산이며, 5일차 종료 후 Final Settlement로 Run 전체 성과를 평가한다.

즉시 실패 조건이 발생하면 Day 정산을 기다리지 않고 Run 전체가 실패한다.
